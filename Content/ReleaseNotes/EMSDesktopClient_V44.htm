Release Notes (Version 44)

EMS release notes for the Version 44 EMS Desktop Client are organized by Month and Year.

To keep your applications up-to-date, download and run the latest database patch, plus the latest builds for your installed applications. Both the database patch and the application builds are cumulative (all previous fixes are included in the latest build and patch we provide). We recommend that all EMS application components are kept up-to-date together.

For an overview of Version 44, see V44 Whats New.

EMS Internal Reference # 16482
(For EMS internal use only)

Enhancement

APIs

 

Additional APIs were requested as a feature enhancement

Added the following APIs to the EMS API library:

  • AddServiceOrder: Allows new service orders to be added to bookings.
  • AddServiceOrderDetail: Allows new service order details (resources) to be added to existing service orders
  • UpdateServiceOrder: Allows for edits to service orders.
  • UpdateServiceOrderDetail: Allows for edits to service order details (resources).
  • DeleteServiceOrderDetail: Allows deletion of service order details (resources).
  • DeleteServiceOrder: Allows deletion of service orders.
  • AddReservation5: Allows reservations and bookings tied to a Web User, including Billing Reference/PO Numbers and setup count. 
  • GetCategories: Retrieves category information.
  • GetCategoryStates: Retrieves categories for the building ID entered.
  • GetResources: Retrieves active resource information for the category entered.
  • GetPricingMethods: Retrieves active pricing methods to use with Service Order.
  • ValidateBilling: Checks that Billing Reference, PO Number, or both are valid.

Enhancement

Reports > Daily > Setup Worksheet

When running the setup worksheet categories were not being displayed unless the “Limit Bookings to those with Details in Selected Categories” was selected.

The Setup Worksheet will now run and show all the selected categories information without limiting to those in selected categories.

Reports > Statistics > Room Utilization

 

When running the report for a period of time that included a Holiday or Building Closure, hours when the building was closed were incorrectly displaying on the report. 

Fix: Holidays and Building Closures are accounted for in the Room Utilization report and no longer display on the report.

Time Zones

 

Brazil has two Time Zones, EMS only accounted for the Brazil Time Zone.

Fix: Brazil Summer Time Zone has been added to EMS to account for both Brazilian Time Zones.

Reports > Daily > Setup Worksheet

 

When running a Setup Worksheet with just UDFs and Notes, the report would have overlapping information.

Fix: The Setup Worksheet will no longer have any overlapping information.

Custom Queries

 

Running a custom query in the EMS Desktop Client without a database server connection caused the error, "a network related or instance specific error occurred while establishing a connection to SQL Server.

Fix: Modified data connections to ensure that this error no longer presents and custom queries successfully complete.

PAM Web Service

 

Application pool errors for PAM were crashing web servers: "The Autodiscover service couldn't be located". These errors occurred when an invalid email address was encountered.

Fix: Modified application pool  behavior so that it no longer causes web servers to crash.

EMS Navigator, VEMS Room Wizard

When users moved a booking with a resource to a room that excluded those resources, they were not being alerted.

Fix: Modified field behavior on the EMS Desktop Client Navigator screen and on the VEMS Room Wizard to ensure that users are appropriately alerted that the room change was performed but that the resource is now unavailable. Resources remain in the booking with Quantity set to zero, highlighted in red to further alert users.

Billing > Invoices > Invoicing

Invoicing reports sorted by Subtotal or Grand Total were sorted incorrectly.

Fix: Resolved report sorting to correctly sort by Subtotal or Grand Total.

Web Security Templates

 

Client Web Security Templates had an "Allow Check-In/Check-Out" option that needed to be simplified to "Allow Check-In."

Fix: Modified the field behavior for Web Security Templates so that the option is simply for "Allow Check-In."

Custom Reports

 

Running custom reports could produce an error due to Daylight Savings Time. 

Fix: Modified how EMS handles Daylight Savings Time for reports so that the error no longer occurs and custom reports execute successfully. 

Service Order Notifications

 

Users were unable to add group email addresses to these notifications. 

Fix: Changed the Service Order Notification to allow group email addresses.

Sending Confirmations Hangs

 

Clicking the Confirm button to send confirmation caused slowness for some users.

Fix: Streamlined system validations to eliminate processing delays so that there is no slowness when users send confirmations.

Booking Check-In

 

This field was unavailable for some users. 

Included a recent fix to this issue in the March build to resolve the issue and ensure that the Booking Check-In field is available when appropriate.
