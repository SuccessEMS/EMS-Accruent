Check In

Streamline Meeting Checkin

Modified functionality to streamline user actions when logging into EMS Kiosk and checking into meetings. Now, when users swipe a badge or otherwise log into the application, they are taken directly to a list of their events; if they have an upcoming meeting within the check-in time frame in a room that requires check-in, they are automatically checked in. Additionally, when users create a meeting from the Check In window, they are automatically checked into the meeting.

EMS Internal Reference #s: 26571 and 26572

New Booking

Enable Meeting Duration To Be Set By Number of Hours or Specific Time

Created two new parameters, "Booking Duration Set by Specific End Time or Number of Hours" and "Number of Hours for Booking," to enable administrators to set Kiosk parameters so that users can specify meeting duration by number of hours rather than by a specific end time. 

EMS Internal Reference # 26262

Web Traffic Security Vulnerabilities

The web application sent non-secure user and session information (cookies) during encrypted (SSL) sessions.

Fix: Added validation to all calls and added verb tampering redirection to prevent non http or Get calls from being made to Kiosk.

EMS Internal Reference # 23979

Translations

Bookings Fail in Slovakian

Users were unable reserve a room using a profile with default translation in Slovak.

Fix: Modified functionality to ensure that bookings succeed using Slovakian translation.

EMS Internal Reference # 23907
