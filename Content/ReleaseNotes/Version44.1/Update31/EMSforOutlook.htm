Combination Rooms

Combo Room Components Showed Incorrectly in EMS for Outlook

After reserving a combo room, the Schedule View and List View showed the component rooms as available even though the parent combo room was shown as reserved. Additionally, booking one of these component rooms would cause a conflict window to appear.

Fix: Modified functionality so that Schedule View and List View show the component rooms as unavailable if the parent combo room has been reserved.

Internal Reference # 39620
(For EMS Internal Use Only)

Recurring Reservation

EMS for Outlook "Send Update" Button Not Updating Database

EMS for Outlook was not updating the time in the database when users modified a meeting time via drag-and-drop in the Outlook calendar and then used the EMS for Outlook Send Update button. If the native Outlook Send Update button was used, the time would update in the database, but there would be a few minutes delay, and users would need to refresh the Reservation Book multiple times in EMS Desktop Client to view the change.

Fix: Modified functionality so that when users change a meeting time through drag-and-drop and click the EMS for Outlook Send Update button, the time will be updated in the database.

Internal Reference # 41171
(For EMS Internal Use Only)

Cancel Meeting

Meetings Canceled in EMS for Outlook Were Not Showing As Canceled in the Attendees' Outlook Calendars

When a single or recurring meeting was canceled in EMS for Outlook, the attendees' Outlook calendars did not show the meeting as canceled even though the organizer's Outlook calendar showed the meeting as canceled.

Fix: Modified functionality so that when a single or recurring meeting is canceled in EMS for Outlook, the organizer's and attendees' calendars show the meeting as canceled.

Internal Reference # 41623
(For EMS Internal Use Only)

Create Reservations

Meeting Invitation Did Not Show As Sent When the Organizer Viewed the Invitation After Clicking Send

Meeting invitations were not showing as sent after the organizer clicked the Send button. When the organizer viewed the meeting invite, Outlook displayed the Delete button instead of the Cancel Meeting button, and Outlook also displayed the message, "You haven't sent this meeting invitation yet."

Fix: Modified functionality so that after the organizer clicks Send on a meeting invitation, Outlook will display the Cancel Meeting button when the organizer views the meeting invite, and Outlook will no longer display the message, "You haven't sent this meeting invitation yet."

Internal Reference # 41829
(For EMS Internal Use Only)

Drag-and-Drop Did Not Update Time for Meetings in Beijing Time Zone

For meetings in the Beijing time zone, when users modified the meeting time using drag-and-drop in the Outlook calendar, the EMS for Outlook add-in did not update the time in the database if the change was made on the same day.

Fix: Modified functionality so that the database updates when users in the Beijing time zone make changes via drag-and-drop to the meeting time.

Internal Reference # 42190
(For EMS Internal Use Only)

Performance

Performance Issue with EMS Outlook Add-in

Between Update 29 and Update 30.2, a change was made that caused a performance issue in the EMS Outlook add-in. When users searched for a room, a time zone search call was being sent for every room within the search results instead of only one time zone search call being sent for each building. This change caused the search results to display slowly.

Fix: Restored functionality so that only one time zone call is being sent for each building within the search results and users experience a faster response when searching within the EMS Outlook add-in.
