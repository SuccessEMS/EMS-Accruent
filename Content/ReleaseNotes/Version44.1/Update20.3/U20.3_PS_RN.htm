Enhancements

Area

Description

Combo Room

Filter Can Remove Component Reservations from Users' Bookings

When a combo room is booked, the component rooms are also booked, so they do not display as available. However, we do not want these phantom reservations for the component rooms to display during a search. The filter {includeComponentRooms} has been added to prevent these component reservations from displaying.

Internal Reference # 35041
(For EMS Internal Use Only)

Override Rooms

Filter Can Remove Bookings for Override Description Rooms

The filter {ExcludeOverrideDescriptionRooms} has been added to the booking search calls to prevent these override description rooms from displaying.

Internal Reference # 35299
(For EMS Internal Use Only)

Fixes

Area

Description

DaysAvailable

Incorrect DaysAvailable Value Returned When Range of Days Booking Rule Was Applied

The DaysAvailable value was incorrect and did not reflect the sum of the dates searched minus the conflicts. (For example, if a user is searching for 5 days and a room is booked for one day, the DaysAvailable should be 4.)

Fix: Modified functionality so that the DaysAvailable value reflects the sum of the dates minus the conflicts.

Internal Reference # 34947
(For EMS Internal Use Only)

Booking End Time

Booking End Time Offset Changed After Daylight Savings

If a user created a reservation that started before the Daylight Savings change and ended after the change, the booking's end time was offset.

Fix: Modified functionality so that bookings are created with correct offsets and booking GET returns the correct offsets for start/end times.

Internal Reference # 35046
(For EMS Internal Use Only)

Remove Area Building

Remove Area Building Caused Error Message

Users attempting to delete a building from an area were receiving an error message.

Fix: Modified functionality so that users can remove buildings from areas without an error.

Internal Reference # 35390
(For EMS Internal Use Only)

GET Call

GET Area Building Call Resulted in Error

Users attempting to call the GetAreaBuilding with the area ID and building ID were receiving an error.

Fix: Modified functionality so that users have the ability to look up and return one building in one area.

Internal Reference # 35391
(For EMS Internal Use Only)
