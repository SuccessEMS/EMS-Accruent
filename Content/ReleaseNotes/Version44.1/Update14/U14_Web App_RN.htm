EMS Web App Release Notes V44.1, Update 14

This section describes updates to your EMS software.  

See Also: Contact Customer Support and Updating to the Latest Version of EMS (Patches and Builds).

Name: V44.1 Update 14

Release Date: July 31, 2017

Patch: 43.0.0.201707180

Build #: 44.1.14000.555

 

Enhancements 

Area

Description

My Bookings

My Bookings Includes Join Skype Link

Users who are owners of a reservation with a Skype meeting can now join the Skype meeting by navigating to My Bookings from the homepage. The Join Skype button appears next to the check in/cancel/end now buttons. 

Internal Reference # 29959

My Events

My Bookings Includes Join Skype Link

Users who are owners of a reservation with a Skype meeting can now join the Skype meeting by navigating to My Bookings from the My Events page. The Join Skype button appears next to the check in/cancel/end now buttons. 

Internal Reference # 29961

Process Template Parameter

Unauthenticated User

New EDU Process Template Parameter

Configuration administrators may now modify the taxonomy of the Room Title on the unauthenticated template to reflect the type of space that is available to be requested.

Internal Reference # 31383

Fixes  

Area

Description

Selected Rooms

Unauthenticated Users

Changed Selected Rooms Section to Use Room Title Singular Parameter in an Unathenticated Template

ClassicRequestForm.aspx was not using the Room Title Singular parameter value for this section because an unauthenticated template can only choose a single room.

Fix: Modified functionality so that the Room Title Singular parameter is being used. 

Internal Reference # 31396

Create Reservation

Recurrence Button Was Not Responding When Clicked

Users had to click the recurrence button multiple times to open the start date or start time pop up Window. 

Fix: Modified functionality so that a single click opens the pop-up window as an authenticated user action. 

Internal Reference # 30939

Location Tab

Users Who Clicked Continue Button from Location Tab Did Not Lead to Details Page

Users who tried to click Continue from the Location tab were not being directed to the Details tab. 

Fix: Modified functionality so that user is directed to Details tab when clicking Continue. 

Internal Reference # 28051

Browse Locations

From Browse Locations, Users Were Able to Book Rooms for Templates That Were Unavailable

Process templates that were not enabled for the Web App were still displaying as an option when viewed in Browse Locations or My Account/Personalization. 

Fix: Modified functionality so that templates that are not enabled in Web App do not appear as an option.

Internal Reference # 30166

Selected Time in Browse Locations Was Not Honoring Room Request Time

Authenticated users who select a time from the Browse Locations screen did not see that same time once they were redirected to the Room Request page. 

This bug only occurred if no default time block was set on the process template. 

Fix: Modified functionality so that the time selected on the Browse Locations page persists on the Room Request page. 

Internal Reference # 27195

Reservation Details

Attachments Could Not Be Deleted Once Uploaded in EMS Web App

Users trying to delete attachments were experiencing a spinning wheel that does not advance the screen. 

Fix: Modified functionality so that users can delete attachments. 

Internal Reference # 30576

View Services

Resources Were Entered Under the Wrong Category

When users added a resource to a reservation using the Facility Reservation form, the resource appeared under the wrong category. 

Fix: Modified functionality so that resources are displayed in the correct category. 

Internal Reference # 24006

Adding Rooms

Unauthenticated Users

Users Could Not Add Room Unless Unauthenticated Template Had Categories Selected and Categories Have Groupings

Users cannot add a room unless the unauthenticated template has categories selected. Additionally, the categories themselves must have groupings. 

Fix: Modified functionality so that users can add a room for templates with available categories. 

Internal Reference # 30519

Recurring Reservations

Recurrence Window Was Not Showing Start and End Time Set

The date and time in the Recurrence window were not displaying correctly. 

Fix: Modified functionality so that the start and end time set in the Date & Time window persist in the recurrence window. 

Internal Reference # 30941

Room Search

Unauthenticated Users

Room Search Was Not Following the Attendance Criteria 

When requesting a room as an unauthenticated user, the attendance filter was not being honored by the room search. 

Fix: Modified functionality so that the attendance filter works as expected and the room search results display on those rooms that match the value indicated in the filter. 

Internal Reference # 30942

Service Only Request Template

Service Only Request Templates Were Experiencing Internal Server Errors

When users attempted to select location and date in a Service Only Request process template, they were experiencing an internal server error. 

Fix: Modified functionality so that users can complete their reservation in a Service Only Request process template without receiving internal server errors.

Internal Reference # 30039

Terms & Conditions

Unauthenticated Users

Terms & Conditions Were Not Being Required for Unauthenticated Users

Unauthenticated users entering Request Details and submitting the request were not receiving an error message prompting them to complete the Terms & Conditions. Users proceeding to the Request Details tab from the Service tab should have received a Terms & Conditions error if T&C were required. 

Fix: Modified functionality so that users are prompted when Terms & Conditions are required but not completed. 

Internal Reference # 29178

Create Reservation

Unauthenticated Users

Building Hours Violation Not Working for Unauthenticated Users 

Unauthenticated users who created a reservation with a start time that violated the building's hours of operation did not receive a violation of building hours error message. 

Fix: Modified functionality so that unauthenticated users choosing a start time that violates the building's hours of operation receive an error message. 

Internal Reference # 29832

Resources

Unauthenticated Users

Resources Displayed When Not Available At Checkout for Unauthenticated Users

All resources were displayed as Available at Checkout for unauthenticated users regardless of category configuration. 

Fix: Modified functionality so that unauthenticated users could only view resources that were configured as Available at Checkout. 

Internal Reference # 30359

Account Validation

Link to We Can Send It To You Again Was Broken

Users trying to validate their accounts were receiving an error that their request could not be processed when clicking the "We can send it to you again" link. 

Fix: Modified functionality so that users receive an email that allows them to validate their account. 

Internal Reference # 30513

Create Reservation

Interactive Functions on iPad Were Not Working

When users attempted to create a reservation in the EMS Web App using their iPad, the frame to select location was cutoff and unusable. 

Fix: Modified functionality so that the filter window opens fully in the middle of the screen.

Internal Reference # 29345

Manage Services

Adding Services from Manage Services for One Booking Added Services to All Bookings

Users adding services to one booking within a reservation from Manage Services were finding the services added to all bookings within that reservation. 

Fix: Modified functionality so that the additional services is added only to the booking selected under "Add to Additional Bookings."

Internal Reference # 29501

Server

Unable to Implement Server-Level Encryption FIPS

Fix: Modified functionality so that FIPS can be implemented. 

Internal Reference # 30296

I Know What Room I Want

Apostrophe in Room Name Caused No Results to be Returned in Search

When users added an apostrophe (') when searching in I Know What Room I Want, no results were returned. 

Fix: Modified functionality so that search results are returned if there is an apostrophe in the name. 

Internal Reference # 30651

Add Booking

Add New Booking to Canceled Reservation

Users were able to add new bookings to a canceled reservation. 

Fix: Modified functionality so that new bookings cannot be added to a canceled reservation.

Internal Reference # 27566

Browse Events

Browse Events Filter Incorrectly Displayed Amount of Events

Users viewing the Daily list of events were seeing the correct events, but not the correct number of events. 

Fix: Modified functionality so that the Daily list of events reflects the correct amount of events. 

Internal Reference # 27687

Events Returned from Custom Link Did Not Respect the Number of Events Displayed in Browse Events 

Users who created a custom link to view calendars according to specific views/Buildings (that had more than 50 rooms) could only see the first 50 rooms. All rooms should have been displayed. 

Fix: Modified functionality so that all rooms are displayed via the custom link. 

Internal Reference # 27529

Guests Were Unable to Use the Add to My Calendar Link 

Guest users attempting to add reservations to their calendar using the Add to My Calendar link were receiving the error message, "There was a problem attempting to complete the action you requested. Please try again at a later time."

Fix: Modified functionality so that the .ici file is available for download.

Internal Reference # 29305

Booking Details

European Dates Displayed Incorrectly in EMS Web App

Incorrect dates in Booking Details for users with browsers that have London language and European system time. Date was displayed correctly for Event Date but incorrect for First Booking date. 

Fix: Modified functionality so that dates with European system time display correctly. The First Booking date should display the same date as Event Date. 

Internal Reference # 27970

Add Services

Resources Were Being Grouped Under One Category

Users adding Services from multiple categories to an existing reservation were seeing all services displayed under one category. 

Fix: Modified functionality so that Services display under the appropriate category. 

Internal Reference # 28877

Access

EMS System Access in China

Google fonts were always failing to load when the EMS Web App was being access in China. 

Fix: Modified functionality so that all users in China can access the EMS Web App.

Internal Reference # 29021

Edit Booking

Editing a Booking Alters Setup/Teardown Times

When users edited a booking, the reserved time for the booking reverted back to the default. 

Fix: Modified functionality so that only the changes made should apply to the booking. Changing non-critical booking information will not change critical booking information.

Internal Reference # 26808

Setup Type

Room Capacities Displaying Incorrectly

Users who changed the setup count on the setup type for the room were seeing that the count was not updating with the setup type when searching for available rooms. 

Fix: Modified functionality so that the setup type the count reflects the setup type configured. 

Internal Reference # 26992

Pricing Plan

Resource Pricing Was Not Reflecting the Process Template Parameter

The pricing plan was not following the process template parameter but instead the global system parameter. 

Fix: Modified functionality so that the pricing plan follows the process template. 

Internal Reference # 26995

Edit Booking

Booking Status Resets When an Edit Was Made

When users edited a reserve-type booking in the EMS Web App, the request rooms were being sent back to request status instead of confirmed. 

Fix: Modified functionality so that only the status on the effected booking is changed. 

Internal Reference # 27551

Create an Account

Auto Create Web User Account Function Was Not Working

Auto create web user account function was not working when portal authorization is used. 

Fix: Modified functionality so that auto create web user account function works with portal authorization.

Internal Reference # 24785

Select Services

Resources Could Not Be Added to Booking When Reservation Had 2/3 Canceled Bookings

For reservations that contain 2/3 canceled bookings, resources could not be added. Users received a prompt that "Your selected room(s) and date(s) do not allow for additional Services. Please adjust your selections or continue to the next step."

Fix: Modified functionality so that users can add resources to reservations that contain 2/3 canceled bookings.
