Area

Description

Confirmations

Confirmations Incorrectly Displayed the Booking Date and Start and End Times

For some EMS Desktop Client users, when they generated confirmations, the booking date and the start and end times were displayed incorrectly in the Bookings/Details section. The booking date displayed as %BookingDate% and the start and end time as %TimeEventStart% - %TimeEventEnd%.

Fix: Modified functionality so that confirmations display the actual booking date and the start and end times for a reservation.

Internal Reference # 36803
(For EMS Internal Use Only)

Date Format Error When Specifying Confirmation Settings

When specifying confirmation settings, if users entered the date in the format dd/mm/yyyy instead of mm/dd/yyyy, they would receive the error message “Argument ‘Expression’ is not a valid value.” Also, this issue sometimes prevented users from sending confirmations.

Fix: Modified functionality so that when specifying confirmation settings, the date format dd/mm/yyyy is valid, and users can enter the date in this format without receiving an error message.

Internal Reference # 38836
(For EMS Internal Use Only)

Create Reservations and Confirmations

Error Messages and Incorrect Date When Region Format Was Set to Spanish (Mexico)

When a computer's Region Format was set to Spanish (Mexico), users would receive the error message "... is not a valid time" when they tried to select times for reservations. Additionally, when users generated confirmations, the booking dates were incorrect because the confirmations would list the dates in the mm/dd/yyyy format instead of the dd/mm/yyyy format.

Fix: Modified functionality so that when a computer's Region Format is set to Spanish (Mexico), users can successfully select times for reservations, and users can generate confirmations with the correct booking dates listed (using dd/mm/yyyy format).

Internal Reference # 39351
(For EMS Internal Use Only)

Edit Reservation

Unable to Edit Booking Details When Lock Booking Details If Booking Is Locked Was Set to No

Even though the parameter Lock Booking Details If Booking Is Locked was set to No, users were unable edit the booking details of locked bookings.

Fix: Modified functionality so that when the parameter Lock Booking Details If Booking Is Locked is set to No, users can edit the booking details of locked bookings.
