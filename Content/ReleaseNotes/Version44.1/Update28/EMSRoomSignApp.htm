Fixes

Area

Description

Checking In

VC Reservation Removed from EMS Web App if Users Checked in Via the EMS Room Sign App

For Video Conference (VC) reservations created in the EMS Web App with check in enabled, if users checked into these reservations from the EMS Room Sign App, the reservations would be removed from the EMS Web App. The reservations would still be visible from the EMS Desktop Client.

Fix: Modified functionality so that VC reservations do not disappear from the EMS Web App following check in from the EMS Room Sign App.

Internal Reference # 39831
(For EMS Internal Use Only)

Check In Button Disappeared at Event Start Time Even Though Users Had Not Checked In

The Check In button was disappearing at the event start time even though users had not checked into their meetings. Because the Check In button was no longer showing, users incorrectly assumed they were checked in, and then the auto cancel function would cancel their reservations at the specified auto cancel time.

Fix: Modified functionality so that the Check In button displays at the event time and stays on until the auto cancel function cancels the room.

Internal Reference # 40631
(For EMS Internal Use Only)

Create Reservation

Setting Allow Book Now and Book Later Profile Parameter to No Caused UI Issues

When the Allow Book Now and Book Later profile parameter was set to No in EMS Desktop Client, UI issues occurred in the create reservation modal of the EMS Room Sign App.

Fix: Modified functionality so that there are no longer UI issues in the create reservation modal when the Allow Book Now and Book Later parameter is set to No.

Internal Reference # 38289
(For EMS Internal Use Only)
