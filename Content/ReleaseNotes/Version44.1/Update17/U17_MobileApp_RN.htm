Area

Description

Events List

Start and End Times Were Incorrect in Booking Details for Meetings Occurring After Daylight Saving Time Ends

For meetings that occur after Daylight Saving Time ends, the meeting start and end times were incorrect in the booking details.

Fix: Modified functionality so that when meetings occur after Daylight Saving Time ends, the meeting start and end times will display correctly in the booking details.

Internal Reference # 30910
(For EMS Internal Use Only)

Scan QR Code

After Scanning a QR Code, Selecting the "About This Location" Button Crashed the EMS Mobile App

After scanning a QR code to view a room's schedule, the EMS Mobile App would crash if users selected the About This Location button from the room’s schedule/list view.

Fix: Modified functionality so that after scanning a QR code to view a room's schedule, the About This Location page opens if users select the About This Location button from the room’s schedule/list view.  

Internal Reference # 33113
(For EMS Internal Use Only)

After Scanning a QR Code, Schedule/List View Was Shown Even Though Room Was Not Assigned to User Templates, and EMS Mobile App Crashed If Users Tried to Book the Room

After scanning a QR code for a room that was not assigned to any of the users’ templates, the room’s schedule/list view was shown, and if users selected the + button to book the room, the EMS Mobile App would crash.

Fix: Modified functionality so that when users scan a QR code for a room that is not assigned to any of the users’ templates, the EMS Mobile App will not crash; instead, the app will detect that the room is not assigned to the users’ templates, and users will receive an error message.

Internal Reference # 33115
(For EMS Internal Use Only)

After Scanning a QR Code, the Room’s Schedule/List View Did Not Load

When scanning a QR code to view a room’s schedule, the room’s schedule/list view did not load unless users re-scanned the QR code or switched between the schedule/list view.

Fix: Modified functionality so that a room’s schedule/list view loads the first time users scan the room’s QR code.

Internal Reference # 33116
(For EMS Internal Use Only)

After Scanning a QR Code, Selecting the + Button in Schedule/List View Crashed the EMS Mobile App

After scanning a room’s QR code, the EMS Mobile App would crash if users selected the + button from the room’s schedule/list view.

Fix: Modified functionality so that after scanning a room’s QR code, the app will not crash when users can select the + button from the room’s schedule/list view; instead, a list of available templates will appear when users select the + button.

Internal Reference # 33117
(For EMS Internal Use Only)

When Scanning a QR Code Associated with a Fully Defaulted Template, Users Experienced Infinite Loading When Trying to Create a Booking

When scanning a QR code associated with a fully defaulted template, users experienced infinite loading instead of having the booking created.

Fix: Modified functionality so that when users scan a QR code associated with a fully defaulted template, the booking is created.

Internal Reference # 33393
(For EMS Internal Use Only)

For Bookings Requiring Check In, Users Weren't Checked In When Scanning the QR Code

For bookings created with rooms that require check in, users were not checked in when they scanned the QR code to check in; instead, they  were shown the events list.

Fix: Modified functionality so that when a booking requires check in and users scan the QR code, users are checked in to the booking and receive a message stating that check in was successful.

Internal Reference # 33396
(For EMS Internal Use Only)
