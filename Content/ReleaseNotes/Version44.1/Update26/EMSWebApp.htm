Reservation Details

Access to Browse Reservation Details

A new role, View Reservation Summary, has been added to everyday user security templates (accessed in the EMS Desktop Client through Configuration > Everyday User Applications > Everyday User Security Templates > New/Edit template > Roles tab).

This new role relates to the reservation summary links in notification/confirmation emails. When a user clicks on a reservation summary link in an email, the EMS Web App will perform an authentication check before displaying the Reservation Summary page: 

  • If the user is the owner of the reservation, or a delegate of the owner, the user can view the Reservation Summary page, even if the user has not been assigned the View Reservation Summary role.
  • If the user has been assigned the View Reservation Summary role, then the user can view the Reservation Summary page.
  • If the user has not been assigned the View Reservation Summary role, then the user cannot view the Reservation Summary page.

Administrators can use this new role to limit access to reservation information from forwarded emails.

See Also: EMS Desktop Client Release Notes Update 26 and Assigning Roles to Users.

Attendee Search

Minimum Character Length for Attendee Search

A new parameter, Minimum Characters to Initiate Type Ahead Search, was added that affects the Find Attendees search field in calendar-enabled templates (Exchange and G Suite). The parameter lets administrators configure a minimum attendee lookup search length. The default is 3 characters.

This new parameter enables administrators to specify a longer duration after the last keystroke entry to prevent performance issues when returning attendee results. Updates recommended only for organizations with complex Exchange or G Suite deployments.

See Also: EMS Desktop Client U26 Release Notes and Everyday User Applications Parameters

Internal Reference # 38480
(For EMS Internal Use Only)

Configuration for Search on Type Ahead Fields

A new parameter, Delay for Type Ahead Search Trigger (ms), was added that affects the Find Attendees search field in calendar-enabled templates (Exchange and G Suite). The parameter lets administrators configure the duration after the last keystroke entry in a search field before triggering a search call for attendees. The default is 400 ms.

This new parameter enables administrators to specify a longer duration before triggering a search event to prevent performance issues when returning attendee results.

See Also: EMS Desktop Client U26 Release Notes and Everyday User Applications Parameters

Create a Reservation

Users Could Request Parts of and Whole Combo Rooms

When Requesting Rooms, users were able to select a combo room and its components without receiving a warning message regarding conflicts.

Fix: Modified functionality so that users can either select the combo room components or the whole combo room. Users will receive an error if they try to do both.

Internal Reference # 36802
(For EMS Internal Use Only)

Responsive State

Reduced Menu Functionality

When viewing the EMS Web App on a small screen in responsive state (on a mobile device), the reduced menu that appears in the upper left-hand corner (also referred to as a hamburger menu) was auto expanding. This made working with the event impossible.

Fix: Modified functionality so that the hamburger menu does not auto expand and users can access their events on the EMS Web App in a responsive state.

Internal Reference # 36862
(For EMS Internal Use Only)

Create a Reservation

Booking Rule Error

EMS Web App users received a booking rule violation error when booking a building that is closed on the current day, even if the day of the request is open.

Fix: Modified functionality so that users can create a reservation on a day the building is closed as long as the building is open on the day of the reservation and as long as the booking complies with all booking rules, such as advanced notice.

Internal Reference # 37663
(For EMS Internal Use Only)

Browse Locations

Reverted Date in Browse Locations

Users were experiencing a reverted date when hitting enter in Browse Locations if the cursor was in the Find a Room search field.

Fix: Modified functionality so that users can complete their search without the date changing.
