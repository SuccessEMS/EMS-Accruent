Security

New Security Role Added to EMS Desktop Client: View Reservation Summary

A new role, View Reservation Summary, has been added to everyday user security templates (accessed in the EMS Desktop Client through Configuration > Everyday User Applications > Everyday User Security Templates > New/Edit template > Roles tab).

This new role relates to the reservation summary links in notification/confirmation emails. When a user clicks on a reservation summary link in an email, the EMS Web App will perform an authentication check before displaying the Reservation Summary page: 

  • If the user is the owner of the reservation, or a delegate of the owner, the user can view the Reservation Summary page, even if the user has not been assigned the View Reservation Summary role.
  • If the user has been assigned the View Reservation Summary role, then the user can view the Reservation Summary page.
  • If the user has not been assigned the View Reservation Summary role, then the user cannot view the Reservation Summary page.

Administrators can use this new role to limit access to reservation information from forwarded emails.

See Also: EMS Web App Release Notes Update 26 and Assigning Roles to Users.

Create/Manage Reservations

Minimum Character Count for Attendee Search

A new system parameter, Minimum Characters to Initiate Type Ahead Search, allows administrators to set the minimum character count to initiate type-ahead searches for the Find Attendees field on calendar-enabled template reservations. The default minimum character count to initiate the attendee lookup is set to 3 characters.

System Parameter
Area: Create/Manage Reservations
Name: Minimum Characters to Initiate Type Ahead Search
Value: Positive Integer
Default Value: 3
Applies to EMS End Points: EMS Web App

See Also: EMS Web App U26 Release Notes and Everyday User Applications Parameters

(For EMS Internal Use Only)

Configure Type-Ahead Search for Attendees

A new system parameter, Delay for Type Ahead Search Trigger, allows administrators to set the duration after the last keystroke entry in a search field before triggering the search call for attendees on calendar-enabled template reservations.

System Parameter
Area: Create/Manage Reservation
Name: Delay for Type Ahead Search Trigger (ms)
Value Type: Drop down selection - 400 ms; 600 ms 800 ms; 1000 ms
Default Value: 400 ms
Applies to EMS End Points: EMS Web App

See Also: EMS Web App U26 Release Notes and Everyday User Applications Parameters
