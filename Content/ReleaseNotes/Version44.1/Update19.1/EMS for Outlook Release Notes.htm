EMS for Outlook Release Notes
V44.1, Update 19.1

This section describes updates to your EMS software.  

See Also: Contact Customer Support and Updating to the Latest Version of EMS (Patches and Builds).

Name: V44.1, Update 19.1

Release Date: December 15, 2017

Patch: 44.1.20000.201712110

Build #:  44.1.19001.1815

Enhancements

Area

Description

User Defaults

Support for Template Default and Personalization Settings

Users can now configure template personalization settings in the 'User Defaults' area of the add-in.  These will save to the EMS database and will apply to using the EMS Web App and Mobile App, as well.

EMS Internal Reference # 25965

System Requirements

System Requirements for EMS Web Server

The system requirements for the Web Server have been updated for the EMS Outlook Add-in due to the new Web deployment option. 

EMS Internal Reference # 33291

Deployment Options

Web Deployment for EMS for Outlook

Administrators can use the web deployment option to host the EMS for Outlook add-in on a web server. This deployment option allows Administrators to push the URL to any EMS Administrators so they can download EMS for Outlook without needing Administrative Privileges on their local machine. 

EMS Internal Reference # 33286

Exchange Room Integration (ERI)

Exchange Room Integration Now Supported

Exchange Room Integration (ERI) is now supported in the new EMS for Outlook, for customers who have access to this feature. An ERI-enabled room or resource can be added or removed and the associated email/attendee will be reflected in Outlook/Exchange.

EMS Internal Reference # 31466

Fixes  

Area

Description

  

Room Filter

Users Could Not Type in the Location Box

The Location box on the Room Filters screen displayed a cursor in the drop-down menu; however, users were unable to enter text. Type-ahead functionality was added to the building drop-down menu under the filtering tool.

Fix: Modified functionality so that users can enter text in the Location box. 

EMS Internal Reference # 33711

 

Inactive Rooms Were Appearing in Room Filter

Buildings with all rooms set to inactive were still appearing in the building drop-down under filters. 

Fix: Modified functionality so that only active rooms appear in the building filter. 

EMS Internal Reference # 33890

 

Conflict Resolution

Selecting Skip Conflict Was Not Updated in Outlook Calendar

The location field in Outlook was not being updated correctly when the user dragged and dropped on the calendar and triggered conflict resolution.

Fix: Modified functionality so that skipping conflicts is reflected in the Outlook calendar. 

EMS Internal Reference # 34133

 

Categories

Categories That Should Not Be Visible Were Being Displayed

Some resources and categories that were not set to 'display to everyday users' were appearing within EMS for Outlook.

Fix: Modified functionality so that any resources that are not set to 'available to everyday users' are not visible to users upon create or edit.

EMS Internal Reference # 34225

 

Delegates

Outlook Delegates Were Receiving Error

In some cases, an error was encountered for Outlook delegates when the field used to authenticate everyday users parameter was set to External Reference or Network ID.

Fix: Modified functionality so that delegates attempting to access EMS for Outlook are able to authenticate. 

EMS Internal Reference # 34254

 

Validate Reservation

Validate Reservation Was Called Twice When Saving a Meeting With Attendees

When saving/sending a meeting with attendees, the validate reservation service was called twice causing a slowdown when saving.

Fix: Modified functionality to saving/sending a meeting to enhance the response time.

EMS Internal Reference # 34667
