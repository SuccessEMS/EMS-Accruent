Security

New Security Role Added to EMS Desktop Client: View Reservation Summary

Available through Configuration > Everyday User Security Templates, the new security role allows Administrators to configure unauthenticated access for viewing reservation summary details.

On the (unauthenticated user) Everyday User Security Template, authentication is not required to view the Reservation Summary page. If not selected, viewing Reservation Summary requires an authenticated session of the EMS Web App. For all other Everyday User Process Templates, the user has access to view the Reservation Summary page. If not selected, only Owner or Delegate of the meeting can view all details of the Reservation Summary page. All other authenticated users will have access to the unauthenticated view of the Reservation Summary page.

Create/Manage Reservations

Minimum Character Count for Attendee Search

Through a system parameter, set the minimum character count to initiate type ahead searches for Find Attendees on calendar enabled templates and reservations. The default minimum character count to initial the attendee lookup is set to three (3) characters.

System Parameter
Area: Create/Manage Reservations
Name: Minimum Characters to Initiate Type Ahead Search
Value: Positive Integer
Default Value: 3
Applies to EMS End Points: EMS Web App

Internal Reference # 38462
(For EMS Internal Use Only)

Configure Type-Ahead Search for Attendees

The new system parameter, Delay for Type Ahead Search Trigger, allows the Administrator to set the duration after the last keystroke entry in a search field before triggering search call for attendees on Calendar-enabled templates and reservations.

System Parameter
Area: Create/Manage Reservation
Name: Delay for Type Ahead Search Trigger (ms)
Value Type: Drop down selection - 400 ms; 600 ms 800 ms; 1000 ms
Default Value: 400 ms
Applies to EMS End Points: EMS Web App

Users Could Not Log In When Using Windows Authentication and Connecting with Web Deploy Service

When using Windows Authentication and connecting using the Web Deploy Service, users could not log in.

Fix: Modified functionality so that users can log in when using Windows Authentication and connecting using the Web Deploy Service.
