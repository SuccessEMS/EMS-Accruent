Availability Search

No Rooms Returned in Availability Search Returned Despite a Value > 1 on Max No. Bookings Per Day

When any value was set to > 1, the service/proc was treating the value as 1 for the Max No. Bookings Per Day booking rule. The availability search returned no available rooms if one booking for that template existed on that day.

Fix: Modified functionality so that when the Max No. Bookings Per Day is set to a value > 0, the availability search correctly applies that value.
