Fixes

Area

Description

Timezone

Everyday User Could Not Book Same-day Booking Under Certain Timezone Circumstances

Users booking a room with certain timezone configurations experienced a "Please enter a valid date" message for that day and could not choose the following day.

Fix: Modified functionality so that users can book for any valid time in the timezone that they are currently in, up to one day in advance (the remainder of today and all day tomorrow).

Internal Reference # 35095
(For EMS Internal Use Only)

Exchange Integration

 

Event Not Displayed in the Availability Grid When Editing Exchange Reservation

When editing an Exchange reservation, events were not displaying properly in the Availability grid.

Fix: Modified functionality events are visible in the Availability grid when editing an Exchange reservation.

Internal Reference # 40054
(For EMS Internal Use Only)

Create a Reservation

 

Error Caused by Multi-select UDFs

When creating a reservation on a multi-select UDF template and more than one value selected, users were unable to complete their reservation.

Fix: Modified functionality so that users can complete their reservation.

Internal Reference # 40086
(For EMS Internal Use Only)

Notification Rules

Notification Rules Were Not Firing

When creating a reservation on a template with multi-select UDFs with at least two answers and separate notification rules, the notification rules were not firing.

Fix: Modified functionality so that notification rules fire for each UDF answer.

Internal Reference # 40088
(For EMS Internal Use Only)

Canceling a Reservation

Setup Attendee Count / Validation Error

When canceling a reservation or booking with a room that had two setup types, users received the attendance error, "Please enter a value greater than or equal to 12." 

Fix: Modified functionality so that Setup and Validation (or no validation) are not be invoked when canceling Bookings from the EMS Web App.

Internal Reference # 40814
(For EMS Internal Use Only)

Make a Reservation

Events in Negative Offset Timezone in the UTC Were Not Displaying

Users attempting to book reservations in negative offset timezones in the UTC experienced difficulty finding a room. Existing bookings were not appearing for certain times on the Schedule Grid for buildings in these timezones; therefore, it was unclear which rooms were available.

Fix: Modified functionality so bookings appear on the schedule grid for buildings in (-) offset timezones.

Internal Reference # 41112
(For EMS Internal Use Only)

Edit a Booking

When Editing Bookings on Multi-Timezone Reservations, Time and Grid Display Times Could Correspond to Incorrect Timezones

When editing a booking on a multi-timezone reservation, the Time and Grid displays were not presenting the earliest timezone first; therefore, if users edited the reservation, it could reset the times of existing bookings to the incorrect timezone.

Fix: Modified functionality so that the time and grid correspond to the correct time zone.
