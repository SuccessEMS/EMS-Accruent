When creating a new reservation through the List View, the column titled “Filter Match” displays a Green Bar. If the Green Bar is filled, the result matches all of the filter criteria. If the Green Bar is not completely filled, clicking on the Green Bar will now display the items that are not available for the room for that date and time.

Fixes  

Area

Description

Edit Booking

Error Received When Clearing the End Time Field

 Users received an internal server error when attempting to edit the booking time after initially clearing the 'End Time' field.

Fix: Modified functionality so that no internal server error is received.

Internal Reference # 31395

Skype for Business

 

Users Unable to Integrate Skype for Business on Internet Explorer and Edge

This error was the result of Internet Explorer and Edge settings. This can be resolved in two ways:

  1. The recommended option is to configure the browser to work around this issue by adding both the EMS Web App and the Microsoft Online login page as entries to the Trusted Sites zone.
  2. Change the Protected Mode setting for one of the zones (using the Security tab in Internet Explorer) so that it matches the other. Please note that this change may impact browsing other sites. 

Fix: Modified functionality so that Skype for Business can be integrated on Internet Explorer (IE) and Edge. 

Internal Reference # 32318

Cancel Booking

 

Users Who Canceled Booking Received Internal Server Errors When Using 'Act As' Functionality

 Internal Server Error received when a booking was canceled that included the "act as"functionality.

Fix: Modified functionality so that no internal server error is received.

Internal Reference # 28887

Add Services

 

Default Quantity Did Not Reflect the Setup Count

The quantity did not reflect the Setup Count that was entered when the booking was created. The Estimated Count field was set to the default value. 

Fix: Modified functionality so that the Estimated Count field is populated with the entered Setup Count. 

Internal Reference # 29257

Book Room

 

Users Received Internal Server Error When Booking Room

If Default Setup Type was not set to display to everyday users, internal server error was received when booking a room. The setup type should be defaulted to 'no preference' or 'none' when the selected default is not enabled for everyday user requests.

Fix: Modified functionality so that users can add a room. 

Internal Reference # 31219

Create Customer Link

 

Users Selecting Ignore Everyday User Application Display Setting Altered Selected Items Under Filter Type

Creating Custom Links in the EMS Web App and selecting "Ignore Everyday User Application Display Settings" cleared the filters after being added.

Fix: Modified functionality so that checking the box for Ignore Everyday User Application Display Settings leaves the filter and item selected. 

Internal Reference # 29139

My Bookings

 

Web Users Could Not Check in to a Booking that Appeared in My Bookings

Web users who could view a booking on their calendar received an error when attempting to check in. 

Fix: Modified functionality so that a check in button appears, allowing the Web User to check in. 

Internal Reference # 28587

Browse Locations

 

Setup Times Displayed Incorrectly for Specific Dates

Reservations taking place on or around the time change to or from Daylight Savings for rooms with no setup/teardown times were appearing with a setup time tail when viewed in Browse Locations.

Fix: Modified functionality so that no setup time is displayed for reservations and rooms without setup/teardown times.

Internal Reference # 28232

View Service Availability

 

Ampersands (&) Not Displayed Correctly on View Service Availability Page

 For categories created with ampersands, View Service Availability page was not correctly displaying these ampersands. 

Fix: Modified functionality so that characters display correctly.

Internal Reference # 28054

User-Defined Fields

 

UDFs Were Deleted When Modifying Event Name

Users editing the reservation name on an existing reservation with at least one past booking were experiencing deleted user-defined fields (UDFs) in the EMS Desktop Client. 

Fix: Modified functionality so that UDFs are not deleted from the reservation when updated.

Internal Reference # 28007

Add Resource

 

Page Froze When Clicking Out of Add Resource Modal

If users clicked out of Add Resource modal without clicking the OK button, the screen would lock and no further action could be taken. 

Fix: Modified functionality so that the screen does not freeze when clicking out of the Add Resource modal.

Internal Reference # 27948

Room Details

 

Viewing Room Details Cleared Room Search

When creating a reservation, viewing the room details was clearing the search criteria for the room search.

Fix: Modified functionality so that the search criteria is not cleared after viewing room details.

Internal Reference # 27817

Default Value for Attendance Parameter

 

Rooms Were Not Displaying if Default Value for Attendance Parameter Was Set to 0

The default value for the attendance parameter was not being followed. 

Fix: Modified functionality so that the default value for attendance was being followed. 

Internal Reference # 27536

Web Process Templates

 

Web Process Template Assignment Error on First Login

Users in "activate on login" status that were assigned web process templates that were also flagged as "available to new everyday users" were receiving an error at first login.  

Fix:  Modified functionality so that users were not receiving an error at first login.

Internal Reference # 27089

Room Images

 

PDF Attachments Were Not Opening

PDF floor plans added under Room Images tab were not opening in the EMS Web App. 

Fix:  Modified functionality so that PDFs open correctly in the EMS Web App.

Internal Reference # 27029

Edit Reservation

 

Issues Booking Multiple Rooms on Multiple Days

For bookings occurring on multiple days in multiple rooms, users were unable to add a room at time of initial booking. Users were able to add booking during reservation edit.

Fix:  Modified functionality so that rooms can be added at initial time of booking.

Internal Reference # 27022

Add Resources

 

When a Reservation Had 2/3 Canceled Bookings, Users Were Unable to Add Resources

Users were unable to add services to the last booking of a reservation that had 2 out of 3 canceled bookings.

Fix:  Modified functionality so that users are able to add services to the last booking of a reservation with 2/3 canceled bookings.

Internal Reference # 26806

Reservation Details

 

Users Could Not Successfully Open Reservation Details for a Booking with 1,000+ Bookings

Reservation Details were nearly impossible to open from the Browse Events and Home Page for reservations with a large number of bookings.

Fix: Modified functionality so that a loading animation/overlay displays while the modal data is loading.

Internal Reference # 26912

Booking Date

 

Users Could Not Extend Booking Date Beyond the Last Date Available of Parameter Value

Users could not change the booking date to the last day of the parameter value.  They received an error message stating, “Your booking violates the last allowed booking date. We did not update your booking.”

Fix:  Modified functionality so that Edit Booking is available on the last day of the parameter value.  

Internal Reference # 26727

Add Services

 

Add Services Button Was Experiencing Lag Time and Services Were Being Added Twice

Users were experiencing a lag time when clicking Add Services to a reservation with a high number of bookings.  It was not clear if the services were being added, so users were clicking the button again. This caused services to be added twice.

Fix:  Modified functionality so that users are aware that the changes are being process and should not click the button a second time.

Internal Reference # 25077

Create Reservation

 

After Reservation Was Created, Reservation Summary Displayed a Warning that Resource Was in Conflict

Although users had not added an excluded resource to a room, users were receiving the error message, "Warning: Booking in conflict status or had a resource quantity reset," on the Reservation Summary page. This was happening because the resource was being added to the room with a quantity set to “0.”

Fix: Modified functionality so that when resource exclusions apply to a room, the resource is not added to the booking with a resource quantity of 0. 

Internal Reference # 30619

Browse Events

Unauthenticated Users

 

Users Could Browse Events Not Available on the Unauthenticated Template

When browsing events, unauthenticated users could view events and buildings that were not available on the unauthenticated template.  

Fix: Modified functionality so that unauthenticated users are unable view events and buildings that are not available on the unauthenticated template.

Internal Reference # 30774

Room Request

Unauthenticated User

 

Default Times Did Not Match the Start and End Times Set in the Unauthenticated Template

When requesting a room as an unauthenticated user, the default start and end times did not match the times set in the unauthenticated template.

Fix: Modified functionality so that the default start and end times match the start and end times set in the unauthenticated template. If no default start and end times are set in the unauthenticated template, the start time and end time displays as follows:

  • default start time = current time rounded up to the next half hour 
  • default end time = one hour past default start time

Internal Reference # 30944

Category > Service Type

"None" Option Was Not Possible as a Default Service Type 

When the default service type for a category was set to "none," the Service Type drop-down field showed the first service type from the default list, and the user was not required to view the list and select a service type. 

Fix: Modified functionality so that when the default service type is set to "none," the Service Type drop-down field displays "Select Service Type." The user is then required to select a service type from the drop-down. 

Internal Reference # 30956

Password Assistance

 

Error Message Displayed When Captcha Show Another Code Link Was Clicked

In Password Assistance, users were receiving an error message when they clicked the Captcha Show another code link while trying to recover their password.

Fix: Modified functionality so that Captcha loads another code when users click the Show another code link.

Internal Reference # 31052

Add Services

 

Terms and Conditions Checkbox Not Working as Expected

When a user added a service without first selecting the terms and conditions checkbox, the service would be added on the right side of the screen with no price, and the message “terms and conditions is required for Service Requests” would appear in a red bar. If the user selected the terms and conditions checkbox after the red bar message appeared, the service still showed on the right side of the screen with no price, and the service was not be added to the reservation.

Fix: Modified functionality so that the service can be added to the reservation once the user selects the terms and conditions checkbox after the red bar terms and conditions message appears.

Internal Reference # 31094

Edit Reservation

Delete Icon Missing for Rooms with Setup and Teardown Hours

When editing reservations created in an exchange template, users were unable to delete rooms with setup and teardown hours because the delete icons were missing for these rooms.

Fix: Modified functionality so that the delete icons are available for rooms with setup and teardown hours.  

Internal Reference # 31228

Browser

Configuration

Invalid Browser Version Message for Internet Explorer 11

For Internet Explorer 11, users were receiving an invalid browser version message because the setting for the browser compatibility message was not being retained when the web.config file was updated.

Fix: Modified functionality so that when upgrading to a new version of EMS Web App, the browser compatibility setting in the web.config file will be retained and no longer overwritten and reset to the default setting. 

Internal Reference # 31319

Create Reservation

 

Service Order Time Changes Not Reflected When Booking Rooms from Different Buildings

When booking rooms from multiple buildings (in different time zones) and adding a catering service, if the service time was changed, the service order time changes were not reflecting. These changes were not reflected in the confirmation or in EMS Desktop Client.

Fix: Modified functionality so that the service order times reflect the changes made during the initial reservation.

Internal Reference # 31366

Room Search

 

Floor Plan Drop-Down List Did Not Display Filter Results After the Initial Filter Request

The floor plan filter was only being honored the first time users searched with the filter. If users modified the floor plan filter and searched again, the filter did not reflect the new search, and the floor plan filter drop-down list still showed results from the original search.

Fix: Modified functionality so that the floor plan drop-down list displays the correct filter results when users modify the floor plan filter after an initial search.

Internal Reference # 31527

Create Reservation

Unauthenticated Room Request

Attendance Filter Value Was Not Honored for Rooms with Multiple Setup Types

For reservations created in the unauthenticated template, the search results did not honor the Attendance filter value, and the Room Search Results did not display all the rooms that were available within the search parameters.  

Fix: Modified functionality so that when the validation parameter is set to Setup and Validate, the attendance is validated against the min/max of the selected setup type for each search and the filter results show all the available rooms within the search parameters.

For reservations created in a service only template, the estimated count value was not reflecting the Default Value for Attendee parameter after a resource was deleted.
