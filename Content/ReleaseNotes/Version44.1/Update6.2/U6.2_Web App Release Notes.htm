V44.1 Update 6.2

Name: V44.1 Update 6.2

Release Date: October 26, 2016

Patch: 2016.10.24.0

Build #:  44.1.688.0

Fixes  

Area

Description

My Account > Personalization > Templates

Building Should Default to Building Searched

When booking a new event, the building did not default to the last building the user searched for, as in previous versions of VEMS.

Fix: Modified validations so that the user template personalization updates based on the user's last building search.

23946

Recurring Meetings > Reservation Details > Add Services

Service Category Attendees Not Added to All Recurrences

When adding attendees to a service category for a recurrent meeting, they were only added to the first booking.

Fix: Modified functionality to ensure that adding attendees to a service category affects all bookings for recurrent meetings. 

24178

Service-Only Bookings Incorrectly Adjusted for Time Zone by 5 Hours 

When booking a recurring service using a Service Only template, bookings were created -5h to time zone in which they were created.

Fix: Modified validations so that service-only bookings now use the selected building's time zone at the time specified by the user and times are not auto-adjusted.

24224

New Booking > Search for a Room

Calendar Displays Incorrectly Due to Time Zone Conflict

When the user's time zone differed from the chosen time zone for the room availability calendar, some timelines displayed the wrong times.

Fix: Modified the display to properly adjust to the chosen time zone.

24229

Filtering Locations for Single-Digit Dates Causes Error on Internet Explorer

When booking an event for a single-digit date (such as 01.01.2017) on IE and filtering the Browse Locations page by Location, Capacity, Features, Floors, Room, Room Types, or Setup Types, an error occurred.

Fix: Modified page to load properly and the error no longer occurs.

 24232

Adding a Second Room for Bookings After November 01 Causes Error on Internet Explorer 11

When booking an event after November 01 on IE 11 and adding a second room, the error "Internal Server Error" occurred.

Fix: Modified validations so that users can now add more rooms and the error no longer occurs.
