Enhancements

Area

Description

Parameters

Updated search bookings route

The search bookings route has been updated in order to get bookings changed within a timeframe. The following filters were added to the search bookings route:

  • minDateChanged
  • maxDateChanged

EMS Internal Reference # 29333
(For EMS Internal Use Only)

Authentication

Authentication works with Azure IdP

An enhancement was made that fixed an issue related to SAML and ADFS authentication. Authentication now works using Azure IdP.

EMS Internal Reference # 42145
(For EMS Internal Use Only)

Service Orders

State ID can be set when adding or editing a service order

Users can now set the state ID while adding a service order to a booking or while editing a service order in a booking.

The following routes have been updated to include state Id of service in the request body:

  • POST/bookings/{id}/serviceorders
  • PATCH/serviceorders/{id}

EMS Internal Reference # 42836 and 42838
(For EMS Internal Use Only)

Service Orders

Added new GET Category States route

A new route, GET/Categories/{id}/states, has been added. Users can now get states of the categories while adding a service order to booking.

EMS Internal Reference # 42930
(For EMS Internal Use Only)

Booking Search Call

Added 'HideGroup' room property to booking search call

In the booking search call (/bookings/actions/search), a 'hideGroup' property has been added to the BookingRoomModel being returned. The 'hideGroup' property is used to determine when to replace a group name with the Private label when viewing room details in EMS Web App.

EMS Internal Reference # 43067
(For EMS Internal Use Only)

Skype for Business

Skype for Business Meeting Option Parameter Added

A new meeting option parameter has been added to the EMS Platform Services Admin UI under Skype for Business on the Conferencing tab. The parameter These people don't have to wait in the lobby is a drop-down field with the following values:

  • Only me, the meeting organizer
  • People I invite from my company
  • Anyone from my organization
  • Anyone (no restrictions

The default value is Anyone from my organization.

EMS Internal Reference # 42219 and 43684
(For EMS Internal Use Only)

Fixes

Area

Description

API Calls

Incorrect Group Email Address was returned

When making the POST /bookings/actions/search REST call, the incorrect Group Email Address was returned.

Fix: Modified functionality so that the correct Group Email Address is returned.

EMS Internal Reference # 40001
(For EMS Internal Use Only)

Parameters

Reservation not inserted when AltTempContact parameter exceeded 50 characters

When a value for altTempContact exceeded 50 characters, a reservation appeared to be created but was in fact an orphaned booking.

Fix: Modified functionality so that the parameter field value is validated prior to insertion and an error is returned if the values do not meet the requirements.

EMS Internal Reference # 41134
(For EMS Internal Use Only)

SAML Authentication

 

Signature validation failing (SAML)

Some customers' SAML configurations were not allowing SAML Responses to validate in the EMS Platform (using a Shibboleth IdP).

Fix: Modified functionality so that signatures can be validated.

EMS Internal Reference # 41651
(For EMS Internal Use Only)

API

 

Request Body was missing for Get UDF Values for Multiple Rooms in the Swagger documentation.

In the Swagger documentation, the Get UDF Values for Multiple Rooms call did not include the Request Body section with the RoomIds array variable. The RoomIds array is a required field in the body for the Get UDF Values for Multiple Rooms call.

Fix: Added the Request Body section to the Swagger documentation.

EMS Internal Reference # 43077
(For EMS Internal Use Only)

API

 

'Delete Remove Delegates Response' missing in Swagger documentation

There was incorrect documentation for myuser/delegates/{delegateId}. The Swagger documentation for the Remove Delegates call showed a message array in the response. When calling this API, the response received was a single message not in an array.

Fix: Fixed the response for the myuser/delegates/{delegateId} in the Swagger documentation.
