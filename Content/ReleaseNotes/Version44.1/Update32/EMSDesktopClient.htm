Fixes

Area

Description

Software Updates

Older EMS Database Versions were not auditing the latest ClientConnection in tblServerVersion table

When older EMS Desktop Client versions connected to the database, they were not receiving the alert that a newer version was available. The ClientVersion column in tblServerVersion was being updated with the last version of the Desktop Client used instead of the latest version.

Fix: Modified functionality so that when older Desktop Client versions connect to the database, they receive the alert that a newer version is available.

Internal Reference # 27232
(For EMS Internal Use Only)

Reports

User Defined Fields overlapping categories on setup worksheet

In the Setup Worksheet report, statuses with longer strings created a gap row in report that resulted in overlapping lines.

Fix: Modified functionality so that reports are generated cleanly with no overlapping lines.

Internal Reference # 34370
(For EMS Internal Use Only)

Room Utilization Report

 

Room Utilization Report Not Functioning When Utilizing Special Dates

Room Utilization Report data with Special dates were incorrect. All options pertaining to 'Availability Based On' returned the same incorrect data.

Fix: Modified functionality so that Room Utilization Reports include correct data.

Internal Reference # 37324
(For EMS Internal Use Only)

EMS for Google Calendar

 

EMS for Google Calendar Checkbox

For users with an EMS for Google Calendar license, an 'Enable for EMS for Google Calendar' checkbox should be available on process templates. After checking this box, the tab EMS for Google Calendar Conflicts will appear.

Fix: Modified functionality so that the appropriate checkbox and tab appear for EMS for Google Calendar license holders.

Internal Reference # 42989
(For EMS Internal Use Only)

Audit Tab

Editing Critical Room Information Was Recorded in the Audit Tab As a Change to the Everyday User Process Template

For rooms tied to an Everyday User Process Template, when a user modified critical room information, such as the room code, the user was incorrectly recorded on the Audit tab as changing the Everyday User Process Template.

Fix: Modified functionality so that the Audit tab for Everyday User Process Templates only records changes made to Everyday User Process Templates.

Internal Reference # 40153
(For EMS Internal Use Only)

Modify Booking

Modify Booking Details Security Process Not Allowing Create New Category on Bookings

Modify Booking Details (expert user process template) was not allowing the user to add, edit, or delete the category and resources on any booking if the user also had access to that category, if the user had at least view-only access to the booking room, or if the user had access to the status the booking was in.

Fix: Modified functionality so that the Modify Booking Details security process allows users to add, edit, or delete the category and resources.

Internal Reference # 43009
(For EMS Internal Use Only)

Notifications

VC Service Order Change Notifications were not triggering on booking changes

For video conference reservations, Service Order Change Notifications were not being triggered to targeted recipient when a booking time changed.

Fix: Modified functionality so that notifications are triggered when a booking time changes.

Internal Reference # 40311
(For EMS Internal Use Only)

Error Message

'Number or Reservations' Message Always Displayed Quantity of 1

When deleting a configuration item that has been used, an error occurred with a message that references "1 Reservation(s) reference(s) this Everyday User." The quantity in this message never changed regardless of how many items referenced the object.

Fix: Modified functionality so that the quantity stated in the error message is correct.

Internal Reference # 38974
(For EMS Internal Use Only)

VC Template

Date Change From pre-DST to post-DST Start Date (or vice versa) on a Booking Created on VC Template Changed Start Time of Second Booking

When the date of a booking created on a VC template was updated within Daylight Savings (DST) date, the associated bookings changed to the incorrect time.

Fix: Modified functionality so that when the date of a booking created on a VC template is updated within a DST date, the associated bookings do not change time.

Internal Reference # 43072
(For EMS Internal Use Only)

Editing Reservation

Second Contact Issue When Editing Reservation

When editing a reservation's Group, the second contact did not default to none and information populated incorrectly.

Fix: Modified functionality so that the drop-down re-populates with contacts from the new group (and only the new group).

Internal Reference # 43434
(For EMS Internal Use Only)

Invoice

Invoices Exclude Email Address

"Email Address" and "Event Type" were missing from the invoice header on reprinted invoices.

Fix: Modified functionality so that department fields display when reprinting an invoice.

Internal Reference # 41820
(For EMS Internal Use Only)

Navigator >Billing Information Tab

"Bill to Group" displayed incorrect contacts

When specifying the "Bill to Group" on the Billing Information tab of the Navigator, the contacts listed were for the group assigned to the reservation.

Fix: Modified functionality so that the contact drop-down pulls the correct list of contacts based on the selected Group.

Internal Reference # 41837
(For EMS Internal Use Only)

DateTime

DateTime Audits Inconsistent for EMS Email and Dashboard Notifications

DateTime audits were different depending on where you were in the EMS Desktop Client.

Fix: Modified functionality so that there is consistency with DateTime auditing.

Internal Reference # 42295
(For EMS Internal Use Only)

User Defined Fields (UDFs)

Rounding of UDF Field Type 'Numeric'

The user defined field type 'Numeric' was rounding to the nearest whole number when added to an existing reservation in the Navigator.

Fix: Modified functionality so that the numeric value is saved to the reservation, including the digits to the right of the decimal point.

Internal Reference # 42424
(For EMS Internal Use Only)

EMS for Google Calendar

 

Room Mailboxes Were Not Being Added to the Google Calendar Event If Status Was Changed from Canceled to Confirmed

If the status of an event as part of a multi-room reservation created in EMS for Google Calendar changed from canceled to confirmed, the room mailboxes were not being added back to the event.

Fix: Modified functionality so that room mailboxes are added back as an attendee after the status changes back to confirmed.
