LDAP Authentication

 

Intermittent slowness experienced during authentication (EMS Cloud Services Customers)

EMS Cloud Services customers experienced intermittent slowness when authenticating with LDAP.

Fix: Modified functionality to reduce latency during LDAP authentication for EMS Cloud Services customers.

Internal Reference # 38324
(For Internal Use Only)

Schedule View

Bookings spanning overnight were not displayed correctly on Schedule View

The end time for bookings that span a night were not displaying correctly when toggling to the next day on the Schedule View.

Fix: Modified functionality so that when toggling to the next day, the booking end time displays correctly.

Internal Reference # 40289
(For Internal Use Only)

Skype for Business

 

Skype information was not being added to reservations

Users were unable to successfully complete the authentication process due to Skype for Business routing to the wrong pool; therefore, Skype was not being added to reservations.

Fix: Modified functionality so that users can successfully authenticate to the Skype server and are routed to the correct pool.

Internal Reference # 41132
(For Internal Use Only)

View Reservation Summary and Create Custom Link

 

EMS Web App no longer functioned after clicking confirmation hyperlink or custom hyperlink

EMS Web App would stop functioning when authenticate users did the following: 

  • Opened a reservation in edit mode, clicked View Reservation Summary link, and then clicked EMS Hyperlink at the bottom the page.

  • Opened the Create Custom Link page, created a new URL and copied it, and then tried to open the link.

This issue was caused by a database change that was being initiated when users accessed certain Web App pages. During the database change, the new EMS Auth Token was not maintained, which caused the application to stop working.

Fix: Modified functionality so that the EMS Web App functions properly after users click a confirmation hyperlink or custom hyperlink.
