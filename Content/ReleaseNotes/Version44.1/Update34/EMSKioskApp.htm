Locate a Person

Updated messaging on Locate a Person

To prevent confusion, the message that displays after users select a person on the Locate a Person page has been updated. The message now reads Locate "xPerson" instead of Continue as "xPerson."

Internal Reference # 42215
(For Internal Use Only)

Make a Reservation

Room search feature filter added to Make a Reservation page

A feature filter has been added to the Make a Reservation page. Users can now select the Feature filter icon and select one or more room features (such as single monitor and dual monitor) to narrow the room search results.

Feature Filter Icon

When using the feature filter, results are displayed as follows:

  • Only rooms matching the selected feature filters are displayed on the floor map.

  • When switching from the floor map to the list view, the same rooms, which match the selected features, are displayed.

  • If no rooms meet the feature filter criteria, the message "No rooms meet your search criteria" will appear.

Internal Reference # 41606, 42200, EMS-1394
(For Internal Use Only)

Today's Events

Parameter added to display or hide the feature filter

A new parameter—Display Features Drop Down—has been added to the Today's Events area for the EMS Kiosk App. This parameter controls whether the new feature filter is displayed in the EMS Kiosk App. The values for the parameter are Yes/No, and the default is Yes. If No is selected, the feature filter will not be displayed.

Users can access the Display Features Drop Down parameter in EMS Desktop Client through the EMS Kiosk global or profile parameters.

Internal Reference # 43247
(For Internal Use Only)

Area

Description

Private Bookings

Private Functionality Not Working

For bookings with an event type, status, or room that was configured with Display to Everyday Users unchecked, booking information was displayed and not shown as "Private." 

Fix: Modified functionality so that booking information does not display when using private functionality.

Internal Reference # 34531
(For Internal Use Only)

Today's Events

Today's Events Not Loading

Reservations created in the EMS Kiosk App in a profile-level status were not displaying in the EMS Desktop Client.

Fix: Modified functionality so that users can successfully create a reservation in a profile-level status that is different than global-level status.

Internal Reference # 41046
(For Internal Use Only)

Floor Plan

 

Floor Plan icons were not aligned

After upgrading to Update 31, the EMS Floor Plan icons were misaligned for some EMS Kiosk users.

Fix: Modified functionality so that the EMS Floor Plan icons are correctly aligned.

Internal Reference # 43120
(For Internal Use Only)

Logging Out

Selecting the Main Menu button was not ending user sessions

When the parameters Require Authentication for all Kiosk Functions and Require Timeout/Logoff to Terminate User Session were both set to No, EMS Kiosk App was not ending user sessions when the Main Menu button was selected from the Check-in Confirmation page. The End Session button was displayed in the upper-right of the screen after users checked in and selected Main Menu.

Fix: Modified functionality so that when the parameters are set as described above and users select the Main Menu button on the Check-in Confirmation page, the user sessions are ended.

Internal Reference # EMS-1397
(For Internal Use Only)
