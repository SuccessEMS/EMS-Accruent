V44.1, Update 19

This section describes updates to your EMS software.  

See Also: Contact Customer Support and Updating to the Latest Version of EMS (Patches and Builds).

Name: V44.1, Update 19

Release Date: November 30, 2017

Patch: 44.1.19000.201711170

Build #:  44.1.19000.242

Enhancements

Area

Description

Profile

Profile Retrieved at 30-minute Intervals Only in Idle State

The EMS Kiosk App will refresh and reapply profiles only when in an idle state.  Users will not experience the EMS Kiosk App being forced into an invalid state to retrieve profiles while trying to use the app. 

Internal Reference # 31781

Building Hours

If a Building Is Closed with Web Only Option, it Will Be Considered Open in the EMS Kiosk App

If the building hours for the building are set to Closed (Web Only), then the building will be open in the EMS Kiosk App.

Internal Reference # 32424

Check In

Auto-check in Reservations Can Now Be Created Within the Check in Window

 Creating a reservation in the Check In Window will automatically check the user into the meeting. 

Internal Reference # 33097

Locate a Person


Make a Reservation

Loading Screens Allow Users to Locate a Person and Make a Reservation

These loading screens allow users to locate a person and make a reservation. All profile changes are retrieved when the screen is loading so that users see the most updated information.

Internal Reference # 33640

Language Translations

Language Can Be Changed Throughout the Entire EMS Kiosk App

Users can reset selection on the change language page if it has been changed since page load.

Internal Reference # 33753

All Dates and Times Can Be Converted with EMS Kiosk App Translation

Whenever a language is changed in the EMS Kiosk App, dates and times update accordingly. All dates and times throughout the EMS Kiosk App follow local formatting/language for the language that has been selected.

Internal Reference # 33857

Fixes 

Area

Description

Make a Reservation

Combo Room Availability Not Working

Combo room availability on Make a Reservation does not work correctly. Combo rooms and components were not displaying proper availability on Make a Reservation. Rooms that were booked were displaying as available.

Fix: Modified functionality so that combo rooms and components display the proper availability on Make a Reservation.

Internal Reference # 32334

Locate a Person > Find Nearby Space  > Authentication 

Validation Message Only Appeared the Second Time Users Clicked Continue

Users authenticating from Find Nearby Space and entering invalid credentials, experienced a validation message only after clicking the Continue button twice. 

Fix: Modified functionality so that the validation message appears. 

Internal Reference # 34222

Display on Web

"Display on Web" Flags Were Being Ignored on the EMS Kiosk App

Users who created an event type, room, and status that was set to not display on web, were seeing the event in EMS Kiosk App but not in the EMS Web App. The event should not be visible in the EMS Kiosk App. 

Fix: Modified functionality so that the event is not visible in the EMS Kiosk App. 

Internal Reference # 33990

Make a Reservation

Reserve and Map Button on Make a Reservation Page Were Not Working in Internet Explorer (IE) Browser

The reserve and map buttons in the List View and the indicator on the floor plan in the Map View were not working in the IE browser. 

Fix: Modified functionality so that users can reserve the room when clicking on the map or reserve buttons.

Internal Reference # 34223

Auto Check In Was Not Working Properly

Users were allowed to automatically check in regardless of whether Allow Check In From Kiosk was set to No. Status included "Checked-in" due to a disabled parameter.

Fix: Modified functionality so that users cannot automatically check in if Allow Check In From Kiosk is set to No.

Internal Reference # 34387

Floor Map

Date/Time Pickers Were Appearing Behind the Floor Map in Chrome

The date/time picker was opening in behind the floor map.

Fix: Modified functionality so that the date/time picker opens in front of the floor map.

Internal Reference # 34385

Make a Reservation

Bookings Were Not Displayed for Groups with Bookings

Groups with bookings were experiencing errors after entering the Group name. Users were experiencing a loading spinner or an empty screen.

Fix: Modified functionality so that bookings are shown after the Group name has been entered.

Internal Reference # 34410

My Reservations

No Bookings Were Shown in the My Reservations Grid

Internal Reference # 34411

Users Could Not Authenticate by Entering Non-unique Group Names

Groups list were not shown after clicking the "Search" button. Entering a unique Group name automatically logged users in.

Fix: Modified functionality so that groups list are shown if the incomplete matching value has been entered.
