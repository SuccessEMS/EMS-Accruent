Create Reservation

 

New Booking Cutoff Enhancements

When users tried to book a location at a time/date that was prior to the New Booking Cutoff time specified in the Everyday User process template, users received the error message “First available booking date violation.” To provide users more information and guidance with their booking cutoff errors, the error message reporting and functionality have been modified for Range of Days and Specific Date cutoffs.

  • Range of Days—When users select a Range of Days that is prior to the New Booking Cutoff time, users will receive the following error message:

“Online Reservations must be made at least X hour(s) (or X day(s)) in advance. Please select a later date/time.”

In the error message, X hours and X days are dynamic to the lead time established by the New Booking Cutoff time parameter.

  • Specific Date—The following enhancements were made:
  • In the calendar selector, users are unable to select an invalid date.
  • If users type in an invalid date, they will receive an inline message that tells them to select a valid date.
  • If a selected date becomes invalid between the time users select the date, and the time users click Create Reservation (or the difference between the selected time zone and building time zone causes an issue), users will receive the following error message:

"Your date/time selection occurs before the earliest date allowed. Please check your selected time zone and/or choose a later date/time."

Internal Reference # 27753

“Booking Added” Message Was Confusing and Changed to “Room Added to Reservation”

The message "Booking Added" has been modified to read "Room Added to Reservation." The "Booking Added" message, which appeared after users added a room to a reservation, was confusing because users could interpret the message to mean that their reservation had been created even though users still needed to click Create Reservation to complete the reservation process. 

Internal Reference # 29297

Browse People

 

Browse People Search Provides Option to View Remaining Results

When a Browse People search returned more than 50 results, the search results list only displayed the first 50 matching results, and there was no option to display the remaining results. Functionality was modified so that users now have the option to view the remaining results when more than 50 search results are returned.

Area

Description

My Bookings

Clicking the Previous or Next Button Several Times Caused a Loop

In My Bookings, clicking the Previous or Next button repeatedly caused a loop.  

Fix: Modified functionality so that an activity indicator will display over the My Bookings area while the bookings are being retrieved. Also, when users click the Previous or Next button repeatedly, the app will keep track of the date and not loop. For example, if the app shows bookings for August 8 and a user clicks the Next button four times, the app will show the bookings for August 12. 

Internal Reference # 22840

Reservation Details

Assigned Single Event Type in the Everyday User Process Template Did Not Populate the Event Type Field

When users assigned a single event type in the Everyday User process template, the Event Type field in the Reservation Details tab was empty and did not populate with the assigned event type.

Fix: Modified functionality so that when users assign only one event type in the Everyday User process template, the Event Type field populates with the assigned event type.

Internal Reference # 22979

New Reservation > Attendees

Visitor Checkbox Was Not Checked Even Though Default Was Set to Checked

In the EMS Web App, the Visitor checkbox on the Add New Visitors Attendee screen was not checked even though the Attendees Are Visitors checkbox was selected in the EMS Desktop Client.  

Fix: Modified functionality so that when the Attendees Are Visitors checkbox is selected in the EMS Desktop Client, the Visitor checkbox will be checked in the EMS Web App.

Internal Reference # 23001

Users with pending status could log in and view the EMS Web App; however, the users were unable to create reservations or make requests since their status was pending.

Internal Reference # 31576

Custom Links

When users generated a custom link, the custom link results showed canceled and unapproved events as approved.

Internal Reference # 31352

Custom Links Using Group Filter Experienced Delay When Toggling Monthly View

When viewing reservations through custom links using the Group Filter, users were experiencing a 13-15 second delay when they toggled the monthly view, even if there were no results to display.

Internal Reference # 32118

Create Reservation

 

Wildcard Searching Not Possible in Billing Reference and Purchase Order Number Fields

When creating a reservation using a template that prompted users to enter a Billing Reference (BR) or Purchase Order (PO) number, users had to enter the first two characters of the PO or BR number in the search field to look up the number.

Fix: Modified functionality to allow wildcard searches. When users are prompted to enter a PO or BR number, users can search for a PO/BR number as follows:

  • To search all numbers, users can enter ** in the Billing Reference Number or PO Number search field, and the field will display up to 100 results. The results list shows both the BR/PO number and its description.
  • To perform a single character search, users can enter * and the first character of the number or first character of the description for the BR/PO number. For example, entering *4 will return the first 100 search results for BR or PO numbers that start with the number 4.

Internal Reference # 28924

Reservations Created After User Session Timed Out

Users were able to create reservations after their sessions had timed out. However, users were unable to access these reservations because the reservations were created with an unauthenticated user webuserid (logged as webuserid=-1).  

Fix: Modified functionality so that when sessions time out, users will receive an Internal Server Error message, or they will be redirected to the login page with a timeout message.

Internal Reference # 28961

Service Order Times Did Not Adjust to Building Time Zones

When creating a reservation with service orders in buildings located in different time zones, the service order time was defaulting to the “Make booking in this time zone” time and not adjusting the time for the building’s time zone.

Fix: Modified functionality so that the service order time adjusts to the building’s time zone.

Internal Reference # 30863

All Selected Categories Available at Checkout Checkbox Not Respected

When users created a reservation in EMS Web app, the Services tab was displayed even though the All Selected Categories Available at Checkout checkbox was not selected on the Categories tab in EMS Desktop Client.

Fix: Modified functionality so that when the All Selected Categories Available at Checkout checkbox is not selected in EMS Desktop Client, the Services tab is not displayed in EMS Web app when users create a reservation.

Internal Reference # 32686

Unable to Access Room Request Page When Using SSL

When creating a reservation in an SSL environment, users were unable to access the Room Request page.

Fix: Modified functionality so that users can access the Room Request page in an SSL environment. 

Internal Reference # 32840

Inactive Web Template Was Marked Inactive But Still Able to Book Against

When creating a reservation using an inactive template, users were receiving a server error on submit request/create reservation page. Users should have been receiving an error message alerting them that they cannot access the page. 

Fix: Modified functionality so that users receive an error message alerting them that they cannot access the page when using an inactive template.

Internal Reference # 32218

Location Filter

Filter options available for selection were not displaying only those assigned to rooms available on the template for the selections in the Locations search parameter. This applied to filters on Create, Edit, Browse Locations, and Browse Events.

Fix: Modified functionality so that users selecting a location first can only see the available floors/setup types/room types/features for that location.

Internal Reference # 29041

Maintain Inventory

When adding a resource with a limited quantity to bookings across multiple time zones, the Maintain Inventory parameter was not being respected, and the resource was being added to all of the bookings.

Fix: Modified functionality so that when users add a resource with a limited quantity to bookings across multiple time zones, the Maintain Inventory parameter is respected, the resource is not added to all of the bookings, and users receive the error message “One or More services are unavailable.”
