Fixes

Area

Description

Profile Settings

Cancel and Apply Buttons Were Not Displaying in Portrait or Landscape View

The Cancel and Apply buttons were not displaying in the EMS Room Sign App when changing profile settings.

Fix: Modified functionality so that profile settings window should expand to display the cancel and apply buttons.

Internal Reference # 36651 & 36422
(For EMS Internal Use Only)

Profile/Room Selection

Issue Selecting Profile and Room in EMS Room Sign App

Users were unable to select a profile and/or room that intersected with the time bar within the EMS Room Sign App dropdown list.

Fix: Modified functionality so that users are able to select any room and profile from the dropdown list.

Internal Reference # 36652
(For EMS Internal Use Only)

Start/End Times

End Time Was Not Adjusting Correctly Based on Start Time

End time was not adjusting correctly based on the selected start time, and future bookings were not being created.

Fix: Modified functionality so that end time is adjusted based on selected start time.

Internal Reference # 36653
(For EMS Internal Use Only)

Create a Reservation

Unable to Create Future Bookings

Users could not book a meeting in the future.

Fix: Modified functionality so that users can book a meeting for a later time.

Internal Reference # 36654
(For EMS Internal Use Only)

Settings

Once a PIN Was Entered, the Settings Modal Did Not Save Previous Selections

After entering a PIN, the previously selected profile was not selected in the profile dropdown.

Fix: Modified functionality so that any previously saved settings populate correctly after entering a valid PIN.

Internal Reference # 36655
(For EMS Internal Use Only)

Authentication

Authentication to End Booking Was Required Even if "Authentication to End Booking" Was Set to No

EMS Room Sign App was requiring authentication to end bookings regardless of whether the setting "Require Authentication to End Booking in Progress" was set to "No."

Fix: Modified functionality so that the EMS Room Sign App does not require authentication to end booking if the parameter "Require Authentication to End Booking in Progress" is set to "No."

Internal Reference # 36657
(For EMS Internal Use Only)

Booking Details

Bookings Were Not Recognized As Private

For bookings that were set to private when the Event Type for Private Booking parameter was set to None, the private meeting label was not displaying.

Fix: Modified functionality so that the booking details display.

Internal Reference # 36658
(For EMS Internal Use Only)
