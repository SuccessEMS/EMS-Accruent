Fixes

Area

Description

 

Incorrect Time Bar in Attendee Schedule View

Time labels across the attendees area of schedule view was not reflecting the correct time for the organizer and attendees.

Fix: Modified functionality so that the attendee schedule view reflects the correct time.

Internal Reference # 41582
(For EMS Internal Use Only)

Create a Reservation

Incorrect Booking Rule Error Message

Regardless of what was configured in the "Allow New Bookings Within Range of this Many Days," users experienced the error message, "You cannot book this Room after XX/XX/XXXX. . . ." The date on the error message was one day before the last allowed booking date.

Fix: Modified functionality so that the correct last allowed booking date displays if users try to book beyond that date.

Internal Reference # 41099
(For EMS Internal Use Only)

Cancellations

Canceling One Occurrence in EMS for Outlook Canceled All Bookings

Canceling one occurrence in EMS for Outlook resulted in the cancellation of all dates in the series in the EMS Web App and EMS Desktop Client.

Fix: Modified functionality so that canceling one occurrence does not affect other dates in the series.

Internal Reference # 41533
(For EMS Internal Use Only)

Conflict Resolution and
Recurring Reservations

EMS for Outlook Add-in Was Unresponsive When Users Tried to Resolve Conflicts

When resolving booking conflicts for a recurring meeting, the select room button (+) on the Resolve Conflicts page was unresponsive, and users were unable to select a new location to resolve the conflict. This was caused by rooms configured to alert users where no alert message was actually configured.

Fix: Modified functionality so that the select room button (+) on the Resolve Conflicts page functions properly, and users can select a new location to resolve a conflict.

Internal Reference # 40727
(For EMS Internal Use Only)

Billing Reference and PO Numbers

Group Default Billing Reference (BR) and PO Numbers Were Not Populating

When creating a reservation in EMS for Outlook, the BR and PO values were not populating by default in their respective fields. The defaults were populating when users toggled from a different group back to the original group.

Fix: Modified functionality so that the default BR and PO values for a group are populated in the respective fields by default when creating a new reservation on a template that displays these fields.

Internal Reference # 40968
(For EMS Internal Use Only)

Edit a Reservation

Attendees' Calendars Were Not Updating When Dragging and Dropping

Reservations created using the EMS for Outlook Add-in that were edited by dragging and dropping to a new time were not updating on attendees' calendars.

Fix: Modified functionality so that the attendees' calendars are updated.

Internal Reference # 41827
(For EMS Internal Use Only)

Edit a Reservation

Dragging and Dropping Did Not Update Database

Users were unable to drag/drop a single occurrence in the EMS for Outlook Add-in. The time was not updating in the database unless the time was changed in the meeting drop down.

Fix: Modified functionality so that simply dragging and dropping updates the time in the database.

Internal Reference # 41640
(For EMS Internal Use Only)

Delegates

Edits Made to Delegate Calendar Were Not Reflected in EMS Web App or EMS Desktop Client

Edits made on a delegate's calendar in the EMS for Outlook Add-in were not reflected in the EMS Web App or the EMS Desktop Client.

Fix: Modified functionality so that edits made to a delegate's calendar will be reflected in all EMS access points.

Internal Reference # 40232
(For EMS Internal Use Only)
 

Location Text Field in Occurrence Not Updating

The Location text field in an occurrence was not updating if either the conflict on a recurrence was resolved with the EMS screen or the conflict was resolved on an appointment.

Fix: Modified functionality so that the Location text field updates with room changes.

Internal Reference # 42018
(For EMS Internal Use Only)
