Enhancements

Area

Description

Accessibility

Home Page Functionality Accessible Via Keyboard

Users are now able to tab to and utilize the My Bookings: Date Picker and the Previous, Today, Next selectors using their keyboard.

Internal Reference # 34032
(For EMS Internal Use Only)

Dynamic Filter Accessibility

'Add/Remove' links and multi-select modal functionality are now accessible via the user's keyboard. Users can select all, single, or multiple items from the filter modal.

Internal Reference # 34034
(For EMS Internal Use Only)

Create Reservation

 

Default Start/End Times Enhancement

For both authenticated and unauthenticated templates, when a user changes the date manually, the EMS Web App does not recalculate default times. However, when a user does not change the date manually, the EMS Web App will automatically recalculate the default times.

Internal Reference # 35406
(For EMS Internal Use Only)

Add Services

 

Time Zone Drop Down Added to Applicable Categories

Time zone will now be displayed/used on the Add Services page. This enables users to know the time zone for Service Orders.

Internal Reference # 36339
(For EMS Internal Use Only)

Time Zone Name Added to Services Summary

Time zone will now be displayed on the Services Summary page.

Internal Reference # 36340
(For EMS Internal Use Only)

Service Time Column Added to Booking List

Service Time will now be displayed on the Booking List. This column will display the exact (localized) time at which the Service Order will be added on the Add Services page before actually choosing to add the Service.

Internal Reference # 36341
(For EMS Internal Use Only)

Start/End Time Default Changes with Time Zone Selected

Reservations created with non-VC and non-Exchange enabled templates can now have bookings in different start/end times.

Internal Reference # 37208
(For EMS Internal Use Only)

Reservation Summary > View Services

 

Time Zone Name Added to Applicable Services

Time Zone name will now be displayed next to Service Order times in the View Services section of the Reservation Summary page.

Internal Reference # 36343
(For EMS Internal Use Only)

Security

 

New Security Protocol Parameter Values TLS 1, 1.1, and 1.2

TLS 1,1.1, and TLS 1.2 are options for the "SMTP Security Protocol" parameter. SMTP emails can be sent via the EMS Desktop Client using either TLS 1.1 and TLS 1.2 based on the parameter.

Internal Reference # 36702
(For EMS Internal Use Only)

Reservation Summary > Manage Services

 

Time Zone Added to Manage Services Time Display

Start and End Time now display the local time zone abbreviation of the booking.

Internal Reference # 36823
(For EMS Internal Use Only)

Fixes

Area

Description

Non-VC Template

 

Users Could Not Book from Non-VC Template When Resources Were Booked in Another Room

Users could not book a room that was tied to a resource through a standard NON – VC web process template.

Fix: Modified functionality so that users can make a reservation in a regular process template because the template disregards the Resource inventory.

Internal Reference # 35230
(For EMS Internal Use Only)

Browse People

 

In Mobile Browser, Date Picker Did Not Appear Within View

When accessing the EMS Web App on a Mobile Browser (smaller mobile devices, not tablets), the date picker appeared off-screen when users attempted to change the date on Browse People.

Fix: Modified functionality so that users do not have to scroll to view the date picker on a mobile device.

Internal Reference # 36003
(For EMS Internal Use Only)

Search Button

 

Search Button Was Not Displaying Properly

On iPhone or similar smaller devices, the Search Button was partially displaying as "Sea. . . " instead of Search.

Fix: Modified functionality so that the full button label for search displays.

Internal Reference # 36004
(For EMS Internal Use Only)

Security

 

Vulnerability on the 'Dismissed Mobile' Entity

Users in the mobile version of the EMS Web App were experiencing a vulnerability under the 'Dismissed Mobile' entity.

Fix: Modified functionality so that the ‘Dismissed Mobile’ entity is stored to local storage rather than stored as a cookie. Values in local storage are encoded to protect against XSS attacks.

Internal Reference # 36758
(For EMS Internal Use Only)
