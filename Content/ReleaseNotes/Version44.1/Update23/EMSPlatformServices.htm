Certificates

SP and IdP Certificates Stored in EMS Database

EMS Platform Services stores multiple copies of the IdP public key, SP public key, and SP private key on the database.

Internal Reference # 33860
(For EMS Internal Use Only)

EMS Platform Services Supports the Import of SAML IdP Metadata

Added support for import of SAML IdP Metadata. Service providers can use SAML Metadata XML to generate SAML AuthNRequests to the IdP. This functionality is an alternative to manually configuring Platform Services through the Admin UI SAML configuration page.

The following API call was added: POST /authentication/saml/idpmetadata

Internal Reference # 33828
(For EMS Internal Use Only)

Fixes

Area

Description

Date Completed

 

Users Were Unable to Update Reminder Date Completed Value

Users received an error when updating reminder due date.

Fix: Modified functionality so that users can update the reminder due date without receiving an error.

EMS Internal Reference # 36839
(For EMS Internal Use Only)

Authentication

SAML Authentication Error

After configuring EMS Platform Services and the EMS Mobile App to use SAML authentication, when users logged into the Mobile App, they were re-directed to the IdP, but the IdP gave the following error message: "The request is not a valid SAML2 protocol message."

Fix: Modified functionality so that when SAML authentication is configured in Platform Services and Mobile App, users do not receive the "request is not a valid SAML2 protocol" message, and authentication can occur.

EMS Internal Reference # 37240
(For EMS Internal Use Only)
