Enhancements

Area

Description

Search

 

Search Enhancement

Users can now search Buildings by any user-defined field (UDF). Added UDF filters to the following routes:

  • GET/Buildings
  • POST/buildings/action/search

Internal Reference # 37244
(For EMS Internal Use Only)

User-Defined Fields (UDFs)

 

View UDFs for Multiple Rooms

Users can now view UDFs for multiple rooms IDs.

The following new routes were added:

  • GET/rooms/{id}/userdefinedfields
  • POST/rooms/actions/search/userdefinedfields

EMS Internal Reference # 37245
(For EMS Internal Use Only)

ParentID

ParentID for Attachments and Drawings

A ParentID is returned in the response for the following routes:

  • bookings/actions/search/drawings
  • bookings/actions/search/attachments

EMS Internal Reference # 37928
(For EMS Internal Use Only)

Add Auth Key

Dates Detected From the Certificate

Instead of dates being detected from the Add Auth Key Route, they now come from the certificates themselves.

EMS Internal Reference # 38096
(For EMS Internal Use Only)

Database

Database Enhancement to Accommodate 4096-bit Keys

The database can now fit 4096-bit RSA keys.

EMS Internal Reference # 38100
(For EMS Internal Use Only)

GET Statuses

 

Enhanced GET Statuses Functionality

Additional filters have been added so that GET/Statuses returns all statuses, instead of active statuses and those that have the display on web flag set to true.

EMS Internal Reference # 38130
(For EMS Internal Use Only)

Fixes

Area

Description

SAML Authentication

 

SAML Authentication Was Not Respecting the Identity Provider's Metadata

When importing SAML Identity Provider metadata (with WantAuthnRequestSigned set to false), EMS Platform Services was not respecting the metadata.

Fix: Modified functionality so that when importing SAML Identity Provider metadata (with WantAuthnRequestSigned set to false), EMS Platform Services will not sign authentication requests regardless of whether there are SP keys present in the database.

EMS Internal Reference # 37474
(For EMS Internal Use Only)

Swagger

 

GET Statuses Response Fixed

GET/statues routes returned Booking Model instead of Status Model on Swagger.

Fix: Modified functionality so that users receive the correct response for GET/Statuses route that includes status model.

EMS Internal Reference # 38132
(For EMS Internal Use Only)

Confirmation Emails

 

HTML Tags Displayed in Confirmation Emails

Certain reservations were displaying HTML encoded tags in the body of the HTML confirmation email text.

Fix: Modified functionality so that confirmation emails are rendered without displaying HTML tags.

EMS Internal Reference # 38464
(For EMS Internal Use Only)
 

Correct Time Not Displayed When Updating Booking With Room in Another Timezone

When updating a booking with a room in another building in a different timezone, the correct time was not displaying.

Fix: Modified functionality so that the new booking will have the time set correctly according to its timezone.

EMS Internal Reference # 37064
(For EMS Internal Use Only)

SAML IdP Metadata Import

 

Support of Multiple IdP Certificates

The EMS Platform Services SAML Identify Provider (IdP) metadata import did not support multiple IdP certificates.

Fix: Modified functionality so that the import function supports multiple (possibly concurrently valid) certificates that can be used to validate the signature in the SAML response.

EMS Internal Reference # 37472
(For EMS Internal Use Only)
