Area

Edit Reservation

 

Resources removed from Service Orders

For multi-booking reservations where the bookings have a catering resource and service order, if users edited the reservation and added another resource, but did not apply it to all bookings, and then changed the service type (such as changing from buffet to sit down dinner), the newly added resource would be removed from the service order.

Fix: Modified functionality so that resources are not removed when users change the service type on multi-booking reservations.

Create Reservation > Services > Special Instructions

 

HTML line break tags showed in Special Instructions

HTML line break tags (<br/>) were showing in the Special Instructions that users added to their reservations.

Fix: Restored functionality so that the HTML break tags no longer appear in the Special Instructions.

Availability Search

 

When performing a List View availability search, the setting was ignored for global exact match parameter

For templates using the global setting for Exact Match and that has Set Setup Type Validation Rule set to No Setup and No Validation, when the parameter Only Display Rooms That Exact Match Search Criteria was set to No, which should allow more results to display, the search was returning no results because the parameter Only Display Rooms That Exact Match Search Criteria was being treated as though it was set to Yes, which limits the search results to an exact match of the search criteria.

Fix Modified functionality so that when the parameter Only Display Rooms That Exact Match Search Criteria is set to Yes, the results are not limited to an exact match and "fuzzy matched" results are returned (more search results are returned).

Internal Reference # EMS-5085
(For Internal Use Only)

Create Reservation

 

Blank space(s) were being accepted on Event Name required field making it un-editable from My Events

Users could successfully create a reservation after entering a blank space or spaces (by pressing the space bar) for the Event Name on the Reservation Details page. However, once the reservation was created, users could not edit the reservation because an Event Name is required for opening and editing. Users must click on the Event Name to open the reservation and edit it.

Fix: Modified functionality so that users cannot enter blank space(s) for the Event Name. Users must enter text into the Event Name field before they can successfully create a reservation.
