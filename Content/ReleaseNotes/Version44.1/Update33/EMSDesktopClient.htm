Configuration Menu

Added items for EMS Room Sign App to Configuration menu in EMS Desktop Client

The EMS Desktop Client Configuration menu has been updated to include the following configuration items for EMS Room Sign App: 

  • Deregister Device

  • Global Settings

  • Profiles

  • Refresh All Units

The menu items listed above navigate to the corresponding configuration items available when users click the EMS Glance Settings button (EMS Room Sign) on the toolbar.

The Configuration menu items for EMS Room Sign App will only appear to a user if the following requirements are met: 

  • The user is a system admin or the user is a non-admin who is assigned to the configuration security process for EMS Room Sign App.

  • The database has a license for EMS Room Sign App.

Installation and Permissions

 

EMS.Glance.dll moved to EMS Desktop Client installer and EMS Room Sign App configuration permission added to user processes

Previously, the permission to configure EMS Room Sign App was controlled by the EMS.Glance.dll, which had to be manually installed on the computer of each user who needed permission. As of Update 33, the EMS.Glance.dll has been moved to the standard EMS Desktop Client installer package, and the permission to configure EMS Room Sign App is now granted by admins.

Admins can apply the permission for EMS Room Sign App configuration to users and user process templates in the EMS Desktop Client (processes section, Special > Allow EMS Room Sign Administration):

  • With EMS Room Sign configuration permission, users can access the EMS Glance Settings menu button in EMS Desktop Client and all the EMS Room Sign App settings accessed through the button. Additionally, users can now access these EMS Room Sign App settings through the Configuration menu: 

    • Configuration > Room Sign > Deregister Device

    • Configuration > Room Sign > Global Settings

    • Configuration > Room Sign > Profiles

    • Configuration > Room Sign > Refresh All Units

  • Without EMS Room Sign configuration permission, users do not see the EMS Glance Settings menu button, and users cannot access the EMS Room Sign App configuration settings through the menu button or the Configuration menu.

Automated Reports

Reliability improvements to the Automated Reports Service

Improvements have been made to the Automated Reports Service to enhance reliability and help the service process long-running reports more efficiently. Previously, the Automated Reports Service was vulnerable to crashing if a report took too long to process.

(For Internal Use Only)

2-Way Sync

2-Way Sync – Meeting organizer requires Everyday User

With the 2-Way Sync feature, admins can create bookings on behalf of Everyday Users. These bookings will appear in the Everyday Users' Outlook calendars. For these bookings, admins can also invite the room mailbox so that Everyday Users can take advantage of the self-serve Video Conference equipment in the room with minimal effort.

Note that an Everyday User is required for all Exchange integrated reservations made in the Desktop Client.

Internal Reference # EMS-5026
(For Internal Use Only)

2-Way Sync

2-Way Sync – Ability to restrict date patterns to be compatible with Exchange

When using the new 2-Way Sync feature, users can select Enable Exchange Integration in the Reservation Wizard, which will restrict date patterns to those compatible with Exchange (no random date patterns).

Internal Reference # EMS-5662
(For Internal Use Only)

2-Way Sync

2-Way Sync – Ability to toggle between standard and exchange-enabled booking option in Reservation Wizard

With the 2-Way Sync feature, users can toggle between Standard and Enable Exchange Integration booking options in the Reservation Wizard.

If users select the Standard option and then select more than one date from the calendar picker (random date pattern), the Enable Exchange Integration option will be grayed out because random date patterns are not compatible with Exchange.

Custom Reports

Custom Report Error

In Update 27 and later, when running a Custom Report, the data fields would pull data incorrectly. For example, although the Amount field was set to display the result out to four decimal places (e.g., 0.5332), the field was only displaying the result out to two places (e.g., 0.53).

Fix: Updated the Custom Report DLL so that the data fields pull report data correctly.

(For Internal Use Only)

Automated Reports

Changing the time for an automated report updated the last sync time and the report would not run on the day the time was changed

When users changed the time to run an automated report, the report would not run at the updated time on the day the change was made if the report had already been run prior to the time change.

Example: A user creates a saved report that runs daily at 9:00 a.m. Today, after the report runs at 9:00 a.m., the user changes the time to run the report to 11:00 a.m.; however, the report does not run at 11:00 a.m. This issue occurred because the "last sync" time was updated with the report that was run at 9:00 a.m.; therefore, the system considered that the report had already been run for the day. Note that on subsequent days, the report would run at 11:00 a.m. as expected.

Fix: Modified functionality so that when the time is changed for running an automated report, the report runs at the time specified, even on the day the time was changed.
