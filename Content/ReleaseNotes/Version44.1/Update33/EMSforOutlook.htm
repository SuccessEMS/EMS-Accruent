Edit Reservations

Editing a booking after new booking cutoff time triggered conflict resolution

Editing a booking after the new booking cutoff time (specified in the process template) was triggering conflict resolution, and the conflict resolution page had no rooms available for the user to select. This issue was occurring for bookings edited from either Outlook or the EMS for Outlook add-in.

Fix: Modified functionality so that conflict resolution is not triggered when users edit a booking after the new booking cutoff time. Now, when users edit a booking after the cutoff time, an error message about the cutoff time will be displayed.

Create Reservations

Room availability call was using local system time zone instead of Outlook time zone

Because the room availability call was using the local system time zone instead of the Outlook time zone, the following issues were occurring depending on the time zone selected: 

  • Although the meeting time was in the past, EMS for Outlook was showing rooms available instead of displaying the message "Locations cannot be booked in the past."

  • When the meeting time was not in the past, Schedule View was showing the message "Locations cannot be booked in the past."

  • If users changed the time zone through Outlook > File > Options > Time Zone and did not restart Outlook, the meetings created in EMS for Outlook would show the incorrect meeting time/date in the Outlook calendar and EMS Desktop Client.

    For example, if users changed the time zone from MT (Mountain Time) to EET (Eastern European Time) and booked a meeting 8:00 a.m. EET, the meeting would show in the Outlook calendar and Desktop Client at 5:00 p.m. EET.

Fix: Modified functionality so that the Outlook time zone is used for the room availability call, and the EMS for Outlook add-in functions properly with the selected time zone.

Internal Reference # EMS-2161
(For Internal Use Only)

Edit Reservations

Editing occurrence prevented additional edits

For recurring reservations, when users edited a single occurrence by adding a room and changing the time, the following issues occurred:

  • Standard templates: After saving the occurrence, when users opened the occurrence again, the EMS for Outlook add-in did not show the added room on the Selected Rooms or List View tabs. On the Schedule View tab, the added room was shown, but there was no minus sign (-); therefore, users could not remove the room. Note that in EMS Desktop Client, the room was added correctly.
  • Exchange-enabled templates with ERI rooms: After adding a room and changing the time, when users saved the occurrence, they would be prompted for Conflict Resolution, even though there were no conflicts. If users tried updating the rooms through Conflict Resolution, the rooms would not be updated correctly. Also, the other occurrences in series would not show any scheduled rooms in the EMS for Outlook add-in. In EMS Desktop Client, the reservation would also be incorrect, such as all occurrences in the series moving to one day with multiple bookings.

Fix: Restored functionality so that editing an occurrence does not prevent users from making additional edits and so that edits are correctly applied in both the EMS for Outlook add-in and EMS Desktop Client.

Recurring Reservations

Recurrence pattern was not providing an accurate availability value when the recurrence spanned a Daylight Saving Time change

If a recurring meeting spanned a Daylight Saving Time (DST) change, the recurrence pattern for the room did not provide an accurate availability value. The incorrect availability value allowed other users to view the room as available and book the room in conflict status.

For example: A user schedules Meeting A at 10:00 a.m. in Room 10 on for March 8, March 15, and March 22. Later, another user tries to schedule Meeting B in Room 10 for the same time and dates; however, instead of Room 10 being unavailable for those dates/time, EMS for Outlook shows Room 10 as available for 2 out of 3 days. Note that DST was on March 10, which caused Room 10 to show as available on March 15 and March 22.

Fix: Modified functionality so that EMS for Outlook provides an accurate availability value when a recurrence pattern spans a Daylight Saving Time change.

Create Reservations, Delegates

Error message when creating a reservation as a delegate in EMS for Outlook

In the EMS for Outlook add-in, some users were receiving an error message when they created meetings as delegates on another user's calendar. When these delegate users clicked on the EMS for Outlook add-in from within the meeting they were creating, they would receive an error message stating that they did not have access to book on behalf of the user.

Fix: Modified functionality so that users can successfully create meetings on calendars for which they are delegates .

(For Internal Use Only)

Create Reservations

Text being cut off for resource selection items

In the EMS for Outlook add-in, the text was being cut off for resource selection items if the text was more than 23 characters. This issue made some selection items unreadable, and users were unable to determine which items to select for the resource.

Fix: Modified functionality so that the text wraps to the next line, and the checkbox is centered with the resource selection item text.
