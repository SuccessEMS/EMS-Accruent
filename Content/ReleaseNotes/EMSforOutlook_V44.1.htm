Parameter

 

New parameter for adding text to Outlook meeting body when meeting is created from EMS for Outlook

The parameter—Text to append to EMS for Outlook meetings—has been added. This parameter adds text to Outlook meetings when the meetings are created using EMS for Outlook.

You can access the new Text to append to EMS for Outlook meetings parameter through EMS Desktop Client from System Administration > Settings > Parameters > Everyday User Applications tab. When configuring the parameter, the following apply: 

  • You can enter plain text with a 50 character limit; HTML is not supported. The text you enter will be added to the meeting body.
  • If you leave the parameter's value field empty (Null), no text will be added to the meeting body.

Notifications and Conflict Resolution

 

Reservations for rooms with setup times specified resulted in a conflict resolution loop

EMS Outlook add-in users were experiencing a conflict resolution loop (users could not exit conflict resolution) for reservations made in the EMS Web App on rooms with setup times specified. Users were experiencing the conflict resolution loop when opening the reservations or when trying to dismiss reminders for the reservations.

For example, in the EMS Web App, a room that requires approval is booked for an event with a 30-minute setup time specified. Later, an EMS Outlook add-in user opens the Outlook application 15 minutes before the event starts (the setup time has already begun). Outlook displays a reminder notification for the event, and the user tries to dismiss the notification. However, instead of dismissing the notification, the user is prompted for conflict resolution, and the user cannot exit conflict resolution.

Fix: Modified functionality so that EMS Outlook add-in users are not prompted for conflict resolution when they open reservations for rooms with setup times specified and/or when they try to dismiss reminders for these types of reservations.

(For EMS Internal Use Only)

Internal Reference # 43568
(For EMS Internal Use Only)

Notifications and Errors

 

Error occurred in EMS for Outlook when the meeting end time was extended in EMS Desktop Client while the meeting was in progress

For reservations created in EMS Web App using a calendar-enabled template, users in EMS for Outlook were experiencing an error when the meeting end times were extended in EMS Desktop Client while the meetings were in progress.

For example, a reservation with a meeting start time of 10:00 a.m. is created in EMS Web App. While the meeting is in progress, an EMS Desktop Client user extends the meeting end time from 10:30 a.m. to 10:45 a.m. Once the meeting end time has been extended, EMS for Outlook users receive the following error: "This ongoing reservation cannot be edited."

Fix: Modified functionality so that when a booking end time is extended in EMS Desktop Client while a booking is in progress, the end time is updated in EMS for Outlook, and users do not receive an error.
