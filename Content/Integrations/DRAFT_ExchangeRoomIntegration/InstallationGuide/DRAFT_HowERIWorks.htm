How ERI Works

Once rooms and resources have been configured for Exchange Room Integration (ERI), users can add these rooms or resources to a booking, and ERI will integrate the resource to the booking through the Microsoft® Exchange resource (room/equipment) mailbox. 

ERI also provides bi-directional (2-way) syncing—Exchange to EMS and EMS to Exchange. By syncing booking data in both directions, room and resource availability is correctly maintained and displayed in Exchange calendars. Depending on your configuration and the EMS product being used (for example, EMS Web App), ERI can also keep user availability up-to-date in Exchange calendars.

The following sections provide more information about how these sync methods work:

  • ERI from Exchange-to-EMS Sync

  • ERI from EMS-to-Exchange Sync

For information on the types of booking data that can be synced between EMS and Exchange using ERI, see ERI Interoperability Between EMS Products and Exchange.

ERI from Sync

The ERI from sync allows users to book rooms/resources linked to an Exchange mailbox from native Exchange applications, such as Windows Outlook, Macintosh Outlook, Outlook web, and Outlook Mobile. Exchange will push these native bookings and updates to EMS.

Example of Sync

A user creates a meeting in Outlook Mobile. With ERI installed and configured, EMS Platform Services checks if the meeting's room has a corresponding Exchange mailbox linked to it and if it has a room in EMS configured for the mailbox). If the room does have an Exchange mailbox linked to it and if the room is configured in EMS, a booking will be created in EMS.

EMS Platform Services performs all the work for the side of the sync. EMS Platform Services subscribes to the Exchange mailboxes configured in EMS (rooms or resources). If any changes occur to a room or resource linked to an Exchange mailbox, such as meeting being created, edited, or canceled, then EMSPlatform will be notified about the changes and will , will make those changes on the EMS side.

For information on the types of booking data that can be synced between Exchange and EMS using ERI, see ERI Interoperability Between EMS Products and Exchange.

Create a Booking from a Native Exchange Application

When users create a booking in a native Exchange application, the information that can be synced with EMS is limited by type of booking information that native Exchange applications allow. Native Exchange application allow the following booking information to be synced: 

  • Subject (will become Event Name in EMS)

  • Organizer (only syncs organizer if information is in EMS; will not sync attendees)

  • Date/Time

  • Location

To complete a reservation in EMS, EMS Platform Services needs additional information that is not required in native Exchange applications. This additional information is configured in EMS Platform Services Admin Portal (see Exchange Sync Parameters for more information). When Exchange syncs with EMS, the values selected for the Exchange Sync parameters will be applied to the reservations

From native Exchange applications, four types of reservations can be created:

  • Single meeting with single location (one day, one booking)

  • Single meeting with multiple locations (one day, two or more bookings)

  • Recurring meeting with single location (two or more days, one booking)

  • Recurring meeting with multiple locations (two or more days, two or more bookings)

Modify a Booking from a Native Exchange Application

Users can modify the following meeting information in native Exchange applications and have the information synced and updated in EMS:

  • Subject (Event Name in EMS)

  • Date/Time

  • Room

Cancel a Booking from a Native Exchange Application

When users cancel bookings from native Exchange applications, the information is synced and the booking is canceled in EMS. Also, the room/resource availability is updated and reflected in the room/resource Exchange calendar.

Benefits and Considerations for Using Sync

Many types of work groups use Microsoft® Outlook alone (without EMS) to find and book meeting spaces. Users are accustomed to setting up a meeting in an Outlook or an Exchange-enabled client, inviting attendees, and then clicking on the Rooms button to locate space within their office. With ERI, these types of users can continue to use their non-EMS processes for booking self-service spaces through Microsoft Outlook (referred to as native Outlook booking), and organizations can capture the space utilization statistics of native Outlook bookings in EMS.

Benefits

The following types of users and organizations can benefit from the ERI  Sync:

  • EMS customers who want to maintain existing processes for booking self-service meeting rooms through native Outlook.

  • Organizations with smaller, isolated offices with few meeting rooms or no distinct room booking workflows.

    Rather than give users in these smaller offices a new tool, the users can continue to book spaces in native Outlook applications, and the room booking information will be synced from Exchange to EMS.

  • Organizations that have Macintosh-based work groups/employees who use Outlook for Macintosh to book self-service meeting rooms.

    The ERI Exchange to EMS sync would allow users to continue using their familiar platform and tools. The Mac users could book rooms in Outlook, but the organization would be able to capture and calculate the utilization information for these rooms in EMS.

  • Users with the Outlook mobile app and Outlook Web access would benefit.

    For example, without the ERI sync, if users changed the meeting time/date in the Outlook mobile app, the meeting time/date would change for the users on their Outlook calendar, but the time/date for the booking would not change in EMS. With the ERI sync, ERI would recognize booking edits made from the Outlook mobile app, and after ERI syncs the edited booking, the booking in EMS would be updated. Any conflicts would trigger a configured EMS conflict workflow.

  • Users who book self-service meeting rooms on an impromptu basis, such as using a kiosk or a digital room sign that references a Microsoft Exchange calendar.

    This type of user does not book meetings prior to the meeting time. Instead, the user locates available space and reserves the space using a kiosk or digital room sign. With ERI, these impromptu, on-the-spot bookings would be synced and captured in EMS.

Considerations

The following is a list of considerations and limitations to keep in mind for ERI Exchange to EMS Sync:

  • Organizations with hundreds or thousands of self-service spaces are not good candidates for this functionality.

    Microsoft Outlook offers no filtering capabilities, so users would not be able to filter spaces, like they can in EMS, by location, capacity, type, or room features, and users would need to know the name of the space they want to reserve. In this scenario, an EMS access point would provide a better user experience since users would be able to filter appropriate spaces and save their favorites for quick access.

  • Organizations that have spaces that require an approval are not good candidates for the functionality, as it is designed for self-service meeting rooms only.

    If there are spaces that require an approval workflow, the spaces would have to be managed EMS and outside of Exchange applications. This would create confusion with users because they would have to use different applications and processes to access both types of spaces.

  • Organizations that wish to enforce booking rules would not find the ERI Exchange to EMS sync a good fit.

    While Exchange can provide limited booking rules, it does not provide the level of granularity that EMS provides. If your organization desires greater control on how users can book spaces, a workflow using an EMS client would be more appropriate.

  • Organizations that want to allow the addition of services to a reservation would not find this functionality useful.

    While EMS allows users to include service orders (such as catering, room setup, and audio-visual equipment) with their reservation, the service order feature will not be available in the self-service workflows through Microsoft Outlook.

  • Organizations that want custom information with reservations should not use the ERI Exchange to EMS sync.

    Self-service workflows through Outlook do not provide the ability to specify custom information about a reservation, nor do they allow the ability to assign a billing code or chargeback information to the reservations. If you require billing or chargeback information for spaces, the ERI Exchange to EMS sync is not an appropriate solution.

  • Organizations that require EMS Web App or EMS Mobile App for their users will not be a good fit for this solution. The ERI Exchange to EMS Sync can only support bookings through Outlook or other Exchange clients, such as EMS Room Sign App and EMS Kiosk. Third-party room sign applications that use Exchange resource mailboxes are alternate options.

ERI from Sync

The ERI from  sync allows users to book rooms/resources linked to an Exchange mailbox from EMS applications, and EMS will push these booking updates to Exchange. ERI can perform an sync from the following EMS products: 

  • EMS Desktop Client (new as of Update 33)

  • EMS Room Sign App (new as of Update 33)

  • EMS Kiosk App (new as of Update 33)

  • EMS Web App

  • EMS for Outlook Add-in

  • EMS Mobile App

For information on the types of booking data that can be synced between EMS and Exchange using ERI, see ERI Interoperability Between EMS Products and Exchange.

EMS Desktop Client

As of Update 33, reservations created in EMS Desktop Client can be synced with Exchange using ERI. EMS Desktop Client users can create, edit, and cancel reservations that have rooms linked to Exchange mailboxes. This reservation information will then be pushed to Exchange.

To assist with creating ERI reservations in EMS Desktop Client, Standard and Exchange Enabled radio buttons have been added to the Reservation Wizard. These options appear on the first screen of the Reservation Wizard above the calendar. The default selection is Standard.

  • Standard—This option is for standard, non-Exchange, reservations.

    While Standard is selected, if users choose a random date pattern on the calendar, the Exchange Enabled option is grayed out and cannot be selected. This is done to prevent users from creating reservations with date patterns that are not allowed in Exchange.

  • Exchange Enabled—This option is for Exchange-enabled reservations.

    When Exchange Enabled is selected, users cannot select random date patterns on the calendar because random patterns are not allowed in Exchange. Also, with Exchange Enabled selected, users must select an Everyday User on the final screen of the Wizard. The Everyday User will act as the meeting organizer, which is required by Exchange. The Finish button on the Reservation Wizard is not available until an Everyday User is selected.

EMS Room Sign App

When EMS Room Sign App is used in conjunction with ERI, users can create, edit, extend, end, and cancel bookings that have rooms linked to Exchange mailboxes. These booking updates will then be pushed to Exchange and the changes will show on the organizer's and room's Exchange calendars.

EMS Kiosk App

As of Update 33, you can use Exchange Room Integration with the EMS Kiosk App. To add ERI functionality, enhancements were made in EMS Platform Services to EMS Kiosk routes. To use ERI with EMS Kiosk App, ensure that Update 33 or later of EMS Platform Services is installed.

With ERI installed, EMS Kiosk App users can create, edit, end, and cancel bookings that have rooms linked to Exchange mailboxes. These booking updates will then be pushed to Exchange, and the booking information will be reflected in the organizer's and room's Exchange calendars.

EMS Web App

When everyday users create a booking using an Exchange-enabled template, EMS Web App communicates with Exchange Integration Web Service (EIWS) to enable ERI and EIWS. With the Exchange Room Integration, everyday users can create, edit, and cancel bookings using an Exchange-enabled template. These booking updates will then be pushed to Exchange, which allows organizer and room/resource availability to appear on Exchange calendars. For detailed information on the types of booking updates that can sync from EMS Web App to Exchange, see ERI Interoperability Between EMS Products and Exchange.

Considerations for Using ERI with EMS Web App

When using ERI with EMS Web App, it is recommended that you install Update 33 or later of the EMS Web App software. In Update 33, a feature was added for licensed ERI customers. This feature prevents everyday users from unchecking the Add to Calendar/Send Invitations checkbox on Exchange-enabled templates in EMS Web App. The Add to Calendar/Send Invitations checkbox must be selected to ensure that organizer and room/resource availability appear on Exchange calendars.
