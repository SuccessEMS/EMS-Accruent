Install Exchange Room Integration

To install Exchange Room Integration, complete these steps:

  1. Verify that System Requirements have been met.

  2. Ensure EMS Platform ServicesEMS Platform Services has been installed. Update 33 or later is required.

  3. Ensure the Exchange Integration Web ServiceExchange Integration Web Service is installed.

  4. Install the new License file for ERI. If upgrading, you will need to update your existing license with the license for ERI that has been supplied to you. 

  5. If you will be using ERI to sync from EMS to Exchange with EMS Desktop Client or EMS Room Sign App, you must install the Update 33 version or later of EMS Desktop Client or EMS Room Sign App.

  6. If you will be using ERI with EMS Web App, install Update 33 or later of EMS Web App to ensure the Add to Calendar/Send Invitations checkbox is always selected for Exchange-enabled templates.

  7. Configure 2-way sync deployment for ERI.

  8. Configure your ERI settings in EMS Platform Services Admin Portal as follows: 

    1. Navigate to the EMS Platform Services Admin Portal. 

    2. Select Calendaring > Exchange Sync.

    3. Complete the MS Graph fields as follows: 

      To complete the MS Graph fields, you will need the Application ID, EMS 2-Way Sync Secret, and Directory ID that you copied when you configured 2-way sync deployment (step 7).

      1. In the MSGraphAppID field, enter the Application (client) ID that you copied when you configured 2-way sync deployment (step 6).

      2. In the MSGraphAppSecret field, enter the EMS 2-Way Sync Secret that you copied when you configured 2-way sync deployment (step 6).

      3. and MSGraphTenantID fields, enter the Application (client) ID, Directory (tenant) ID, and EMS 2-Way Sync Secret that you copied when you configured 2-way sync deployment (step 7).

      The values you enter in the preceding MS Graph fields will be stored in dbo/tblRegistry.

    4. In the Exchange Sync Configuration section, set your defaults for the Exchange Sync parameters:

      To properly sync with EMS, reservations created in native Exchange applications, such as Outlook Mobile, must have a Reserve Status, Conflict Status, Cancel Status, Default Group, Reservation Source, and Default Organizer Mailbox (see the following table for details). When Exchange syncs with EMS, the values you have selected for the Exchange Sync parameters will be applied to the reservations.

    5. Click Save.

      Your default parameter selections are saved.
